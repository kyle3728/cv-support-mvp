Custom CUT File

User configurable ASCII file output of part sizes to be used by 3rd party optimizers.

When this Saw NC Link is selected there will be a new Tab in the Machine Catalog named "CUT Fields".

File Properties Area

There are ten general options available to configure your output:

File Name - This field will allow you to specify a specific or partial file name. You may choose to use {JobName} as a variable. Any use of {JobName} will be replaced by the current job.

Example settings:

{JobName}.txt ► Smith.txt

MyOutput.csv ► MyOutput.csv

Out-{JobName}.lst ► Out-Smith.lst

Header - The header is optional. If specified it will be placed once at the top of the file. It can be left blank to omit. Any use of \n will be replaced with a carriage return.

Example Setting: ;This is a heading\nThis is the second line\nThis is the 3rd line.

;This is a heading

This is the second line

This is the 3rd line.

Fixed Width - This will determine if each field will use a specific number of characters in the output. If the actual data has fewer characters then that specified the remaining characters will be padded with space. If the actual data exceeds the character limit then the value will be truncated.

String Length includes the Text Qualifier. If the Field Length is set to 4 and the String is “ABCDEFG” the output will be “AB”. If there is no text qualifier the output would be ABCD.

To set the length of the output field first select the field on the right then enter the Fixed Length at the bottom of the list.

If Fixed Width is false then the Text Length indicates the maximum length of any field. This by default can be left at zero, but can be set to a number it you wish to limit the output length. Unlike Fixed Width, the max length does not include the text qualifier. A max length of 4 with the string ABCDEF would result “ABCD”

Delimiter - This is an optional single character to separate each field. It is commonly a comma (,).

Text Qualifier - This is an optional single character that will surround and text or memo output. Typical values are (“) and (‘).

When the output type is not fixed width if the text qualifier character is found in a text or memo field it will be replaced with either “ or ‘ (whichever is NOT the text qualifier character). The example below uses a double quote (“) as the text qualifier.

A “sample” string with “double” and ‘single’ quotes: “A ‘sample’ string with ‘double’ and ‘single’ quotes.”

Units - The unit can either be Inches or MM. As an option the unit may also be multiplied by a factor of 10, 100, or 1000.

Units Precision - This option specifies the unit precision. Value will be rounded to the nearest selection. The unit precision applies to measurements as well as singles and doubles.

Boolean Format - This option allows for multiple Boolean formats.

Date Format - A string value using (M)onth, (D)ay & (Y)ear as variables. Example settings:

M/D/YY 10/4/13
DD-MM-YYYY 04-10-2013
DDD DD-MM-YYYY Mon 04-10-2013
DDD, MMM DD/YY Mon, Oct 04/13
DDDD, MMMM DD of YYYY Monday, October 04 of 2013

Currency Precision - Similar to unit precision, the value will be rounded to the specified precision.

Field Definition Area

The list on the left are all of the Fields that you can include in your output file. Simply select the fields you would like then click the Right Arrow button to add them to the list on the right. Once a field has been moved over to the list on the right it’s order can be modified by selecting the Up or Down arrow buttons. Multiple fields can be moved at the same time by 1st selecting multiple fields then selecting the Up/Down buttons.

It may be required to output a value that it not represented in the available data list from Cabinet Vision. This type of output is typically not-critical data but is still required by the importing software to fill a space in the output format.
