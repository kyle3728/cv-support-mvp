New Report

This ribbonbar option is used to create a new Report. There are several options in the Advanced Properties window.

Toolbar

Following are the Advanced Property toolbar tools.

Import/Link Data For Report

This tool allows you to import data that is found in databases outside of report.mdb. This can include, but is not limited to, data found in the following databases:

  • Door.ddb

  • BidData.mdb

You can also import data from a database that is not provided by CABINET VISION.

Type - Import

The Import options are as follows:

    • Name - This is the name of the table that will be created, in the report.mdb

    • Source - This is the path to the database that will be used.

    • Table - Once a source is selected, this will list out all of the available tables inside the database to choose from.

    • Fields - This checklist contains the various fields that the selected table has available for import.

Name

As stated above, the name of the import is the name of the table that will be created in the report.mdb file when the report is generated.

Make sure that you follow these simple rules:

      1. Do not use a reserved name. An instance of this would be Parts. There is already a Parts table in the report.mdb and using this as a name will cause the program to not create the table.

      2. Do not use illegal characters. Basically, the safest bet is to only use letters and the underscore (_) character. Anything else may cause problems.

Source

The source is the path to the database. Click on the ellipsis button (…) to open a dialog that will allow you to select a database. Once the database is entered, this field will update to have the proper pathing.

Table

This is a list of tables that are available, in the selected database. Since the import can only create one table at a time, just select the table you want to use here.

Fields

This is a list of the fields that are contained in the database, in the table that was selected. The fields are in alphabetical order, and will be in this order when the table is imported into the report.mdb. Just place a check in the fields that you want imported.

Type - Link

Linking is similar to an import, but rather than creating a new table from a different database, it creates a new 'view' from the report.mdb.

The Link options are as follows:

    • Name - The name of the link as it will appear in the designer.

    • SQL - The SQL statement that will be used to generate the view.

Name

As stated above, the name is what the name of the view will be once it is generated inside of the report.mdb.

Make sure that you follow these simple rules:

      1. Do not use a reserved name. An instance of this would be Parts. There is already a Parts table in the report.mdb and using this as a name will cause the program to not create the table.

      2. Do not use illegal characters. Basically, the safest bet is to only use letters and the underscore (_) character. Anything else may cause problems.

SQL

This is the SQL statement that you can use to generate your view. It must be a valid SQL Select statement.

Create Table Wizard

The Create Table Wizard is a list of the tables, as they will appear in the report for use. The tables can come from multiple datasources, not just the ones provided by Cabinet Vision.

There are nine tabs in the Create Table Wizard:

Source

The Source tab is used to define the data source that this table is drawn from:

This tab has the following options:

    • Name - This is the name of the table, as it will appear in the report.

    • Source - This is the path to the database that will be used.

    • Specify Provider - This allows you the option to define a data source provider [Advanced].

Name

As stated above, this is the name of the table, as it will appear in the report. If you leave this blank the proper table name (from within the database) will be used.

Source

The source of the database, is its location on your hard drive or network. This field is not directly editable, rather, you click on the select source button next to it to browse to your data source.

Specify Provider

There are times when you would like to connect to a non-access database, this allows you to define the provider that will be used to access the data source. Once you have specified the provider, make sure to click the 'Test Data Source' button to make sure it is valid.

The vast majority of reports will never need to have the provider specified, this is for very advanced users that have a multitude of data sources that they want to report from.

Select

The Select tab is used to create a select statement that will pull the appropriate data from the database.

First, you would select the table you want to use, from the drop down list.

Once you have selected the table (or view), you need to add the table. You do that by clicking on the add button.

To know what table(s) (or view(s)) have been added, you can just look back at the tables drop down list and see that a 'plus' symbol has been added to the table name.

When you add multiple tables to a query, you have to specify the join type (or how the data mates together). Look at the Joins section below to see what happens when you add a second table.

Now you will be able to select the fields that you want to pull from. To do this, all you need to do is take a look at the list of columns (if they are greyed out, that means you haven't added the table). You can use + Click or + Click to select multiple fields.

Simply select the column(s) you want, then click on the add columns button.

This will place all of the selected columns into the 'selected columns' list.

Removing a column is as easy as clicking on the column you want to remove (on the right list), and then clicking the remove column button.

Joins

A Join is used to mate the data from more than one table together. You only need to worry about this when you add a second (or more) table to the select tab. When you do this, this dialog is displayed to you.

To understand a join you need to understand how an SQL join works. To that end, it is recommended you visit the following website to learn about SQL Joins.

http://www.w3schools.com/sql/sql_join.asp

Once you understand how a Join works, this dialog will become self explanatory.

Sort

The Sort tab allows you to specify the sorting of the data in the table.

This is a VERY important tab and will have a huge result on the order in which data is displayed in the report. Please spend as much time as it takes to get this correct before doing anything in the report.

With this tab selected, you have the following options available to you:

    • Simple Sort - This allows you to just specify what columns should be sorted, in an ascending fashion.

Simple Sort

This is, as it sounds, a simple way to setup sorting.  Once you select this option, the column list changes as follows:

From here, you would just mark the columns you want to sort by, ascending.

Non-Simple Sort

Once you have determined that you need to sort your columns, you just need select the column and then click one of the sorting buttons:

Sort Ascending:

This button will denote that the column is supposed to be sorted in an ascending fashion.

Sort Descending:

This button will denote that the column is supposed to be sorted in an descending fashion.

Filter

The Filter tab allows you to specify what columns Cabinet Vision can use to filter the data, from inside the Report Center.

When no filter, in the Report Center, has been specified, these will have no effect on the report.

This tab has the following options:

    • Filter by Cabinet ID Field - This allows you to specify the cabinet filtering column.

    • Filter by Top ID Field - This allows you to specify the Counter Top filtering column.

    • Filter by Room ID Field - This allows you to specify the room filtering column.

    • Filter Splash by Part ID Field - This allows you to specify the splash filtering column.

    • Filter By Manage-IT Account ID Field - This allows you to specify the manage-it account filtering column.

Do not set the filters unless the table can actually use this filter. For instance, the Cabinets table would use the Filter by Cabinet ID Field, but not the Filter By Manage-IT Account ID Field.

Filter by Cabinet ID Field

This option allows you to specify what column that Cabinet Vision should use when it has to filter out report items by it's cabinet ID. Just select the column (that was specified in the Select tab) and Cabinet Vision will use this column when you filter by cabinet.

Filter by Top ID Field

This option allows you to specify what column that Cabinet Vision should use when it has to filter out report items by it's Counter Top ID. Just select the column (that was specified in the Select tab) and Cabinet Vision will use this column when you filter by Counter Top.

Filter by Room ID Field

This option allows you to specify what column that Cabinet Vision should use when it has to filter out report items by it's room number. Just select the column (that was specified in the Select tab) and Cabinet Vision will use this column when you filter by room.

Filter Splash by Part ID Field

This option allows you to specify what column that Cabinet Vision should use when it has to filter out report items by it's part id for splash parts. Just select the column (that was specified in the Select tab) and Cabinet Vision will use this column when you filter by splash.

Filter by Manage-IT Account ID field

This option is only for users with Manage-IT.

Cabinet (QTY)

The Cabinet (QTY) tab is a special field tab that allows you to create a cabinet quantity list that is attached to grouped tables.

This tab is ONLY available if you have specified to Group Fields on the Group tab.

This will allow you to create a cabinet quantity list, like this:

1 (2), 2, 3 (2)

This is reminiscent of the cabinet (qty) list found on the old Part_List.rpt file.

This tab has the following options:

Group Cabinet (Qty) - This allows you to select the list type.

Group Cabinet (Qty)

If you have a group of items, let's say parts, that you need to know the cabinet number, as well as the quantity needed in that cabinet, then this is what you would use.

The default value is n/a which means that no list is generated. You can also use one of the following options:

    • Parts

    • Doors

    • Drawers

    • Rollouts

    • Custom

Each one of these options requires a special setup in the Select tab. Essentially, you need to be able to provide a 'Cabinet ID' field, as well as a count field. When specific tables (Parts, Doors, Drawers, Rollouts, respectively) are selected, you will be able to select the appropriate option. The one exception is the Custom option, this is always available and will change the tab to look like this:

With this you now have the following options:

    • CabinetID Field - This is the field that holds the cabinet's ID number.

    • Quantity Field (Count) - This is the field that will be counted.

    • Grouping Criteria - This is a list of the fields that you will group on.

CabinetID Field

This option allows you to select the field that Cabinet Vision will recognize as the Cabinet ID field. For instance, in the Parts table, there is a 'Cabinet ID' field, you would use this field for that table.

Quantity Field (Count)

This is the field that Cabinet Vision will use to generate a count of. Using the parts table as an example, we normally run a count of the 'Part ID' field, so we could select that one.

Grouping Criteria

This is how the records are grouped together. This needs to match how the grouping is setup on the Group tab. If a field is grouped, it should be checked. If it is anything other than 'Group', then it should remain unchecked.

Once you are done with all of the options, Cabinet Vision will use the given criteria to create a the Cabinet (Qty) field and attach it to the appropriate record.

Comment List

The Comment List tab allows you to create a comment list for grouped items.

This tab is ONLY available if you have specified to Group Fields on the Group tab.

This tab has the following options:

    • Comment List - This is the built in comment lists that you can use.

    • Optional Comment List Format - This is the formatting, that you can change, to give you the desired result for the comment list.

Comment List

This is a drop down list that allows you to select one of the built in comment lists that can be created by Cabinet Vision for reporting.

To get one of the automatic items, you must have grouped the fields together properly.  They require the following information:

    • ID field

    • Comment Field

If you don't get an automatic one, because Cabinet Vision might not recognize your query as one for automatic selection, you can still select the Custom option to create your own.

Optional Comment List Format

This allows you to format the comments that are created. You can use variables to setup the list. The variables MUST match a column name that has been selected.

The default value is:

Cab# {Cabinet} - {Comment}

Let's break this down:

Anything that is contained in curly braces is a variable (i.e. {Cabinet}). Anything not contained in curly braces is a string literal (i.e. Cab#). All string literals are displayed, in the report, as they are shown here. All variables will pull the data from the given column name. In this instance, we have the following columns being used:

      • Cabinet

      • Comment

If the columns don't have the same name as here, they you may get a error when you run the report.

Once you select the custom option, the tab changes as follows:

Custom List

This tab allows you to define the values for a list of concatenated values.

To learn more about what concatenation is, please visit this website:

http://en.wikipedia.org/wiki/Concatenation

This feature was specifically designed to allow a concatenated string to be placed on label reports in the S2M Center, but can be used for anything that you can think of in both Cabinet Vision and the S2M Center.

This tab has the following options:

    • Create Custom List - This enables this tab to create a custom list.

    • Primary Field - This is the primary field that represents the unique ID of the data from this table.

    • Foreign Table - This allows you to select the table/view, from the given datasource, that contains the records you wish to concat.

    • Foreign Field - This is the reference to the id of the unique data from this table.

    • Display Field - This is the field that holds the information to concat.

    • Optional Display Format - This allows you to customize the display format for your custom lists.

Create Custom List

Enabling this option allows you to let Cabinet Vision know that you want to create a custom concatenated list.

Primary Field

The primary field is the unique id field that is found in this field.  In the example given here, this would be the Cabinet ID field.

Foreign Table

This is the table, or view, contained in your data source, that has the records that you want to concatenate. In the example given here, this would be the DoorLabels table.

Foreign Field

This is the field, in the selected Foreign Table that holds a reference to the 'ID' field in this table. This data should match the data type, and value, found in this tables Primary Field. In the example given here, this would be the Cabinet D field.

Display Field

This is the field that holds the strings to concatenate together. In the example given here, this would be the DoorLabel field.

Optional Display Format

This allows you to specify a special format for your list.  You can use variables and string literals.

A string literal will be displayed exactly as it is typed here.

A variable will use what's in the curly braces ({}).

You can use one variable as of right now. That's {field}, which will use the 'Display' field that is given. The string literal '\r' is the literal for 'carriage return'. This will create a new line after the string is created.

Relationships

A relationship is a connection, between two tables, that have related data.

For a better understanding of relationships, you can learn about how relational databases (such as Microsoft Access and Microsoft SQL Server) work:

http://en.wikipedia.org/wiki/Relational_database

Do not use a relationship in your report unless absolutely necessary (for sub table creation) as these can cause severe slow down of the report generation.

When you create a new relationship, the following dialog appears:

This dialog has the following options:

    • Primary Table Name - This is a list of the primary tables that have been created for this report.

    • Primary Field Name - This is the list of fields, in the primary table, that hold the 'parent' data.

    • Foreign Table Name - This is a list of the foreign tables that have been created for this report.

    • Foreign Field Name - This is a list of the fields, in the foreign table, that hold the 'child' data.

Primary Table Name

This is the name of the primary table. A primary table holds all of the parent data that will be used. An example would be the Cabinets table in the report.mdb. It has the following fields:

    • Cabinet ID

    • Cabinet Name

    • Width String

    • Height String

    • Depth String

    • Etc.

The Cabinet ID is a primary field as it defines the cabinet record uniquely.

Only tables that have been created, using the Tables interface, will show up in this list.

Primary Field Name

This is the name of the field, found in the primary table, that holds the primary key. As in our previous example (in Primary Table Name) we can see that the Cabinets.Cabinet ID is a unique identifier, so this would be the example that we would use here.

The primary field data MUST have a relational link to the foreign field data.

Foreign Table Name

This is the name of the foreign table. A foreign table holds a reference to a primary table's data. An example would be the Parts table in the report.mdb. It has the following fields:

    • Cabinet ID

    • Door ID

    • Rollout ID

    • Top ID

    • Part ID

    • Etc.

These are all examples of references to other tables' records.

Only tables that have been created, using the Tables interface, will show up in this list.

Foreign Field Name

This is the name of the field, found in the foreign table, that holds the foreign key. As in our previous example (in Foreign Table Name) we can see that the Parts.Cabinet ID is a reference to the Cabinets.Cabinet ID field.

The foreign field data MUST have a relation link to the primary field data.

Add Variable

A variable is an optional place holder to store data that can be used in the report.  The variable is not linked to any table and can be used anywhere in the report.

Variables are dynamic bits of data that can be used to drive formulas in a report. This allows a user to alter the function, style, or other items, of a report, without editing the report itself.

There are default variables that are created by Cabinet Vision whenever you create a report. These variables do not need to be defined by you, the user, they will appear in the report regardless of any intervention on your part.

When you create a new variable, no dialog is displayed. Rather, the variable is added to the list and you can alter it in the property panel on the right of the dialog.

The properties of a variable are as follows:

    • Name - This is the name of the variable, as it will appear in the report designer.

    • Value - This is the value of the variable, as it will appear in the report designer.

    • Type - This is the data type of the variable, this will affect how the report treats the variable.

Name

This is the name of the variable, as it appears in the report. It's how the variable will be referenced in equations and used to display information to the end user.

Value

This is the value that the variable has. You can use this value in equations, or just display it in the report directly.

Type

You can specify one of may types when creating a variable, the list is very large and includes (but is not limited to) the following types:

    • Barcode

    • Text

    • Numeric

    • Boolean

    • Date

    • Drawing

    • RTF (Rich Text Format)

Default Variables

The following variables are created in the report without you creating them:

    • KeyName - This provides the name of the key that is entered into your system. This is a 'Text' variable.

    • RunCounter - This provides you with the S2M Centers run counter, useful when you are creating S2M Center Reports. This is a 'Numeric' variable.

There are also 'Report Variables' that are created for the list and label designer. These are outlined in the designers help file. These are contained in the Variables --> LL folder.

Imports

Imports data from another data source into the report database.  This is only required if you are setting up a table of information that is linking data from an outside source.  If no link is required then there is no need to import, the data can be reported directly from an external source.

Tables

This a gathering of data from multiple locations that all have a 1:1 relation to each other.  Multiple tables can be setup in a report to show different types of data.

Relationships

A relationship is required when you have a 1:E between two tables.  Relationships are slow and should be avoided if possible.  By establishing a relationship you can list multiple records from the foreign table for each record in the primary table.

Variables

A variable is an optional place holder to store data that can be used in the report.  The variable is not linked to any table and can be used anywhere in the report.

Script

Allows the implementation of VBScript into your Reports. VBScript (Visual Basic Scripting Edition) is an Active Scripting language developed by Microsoft that is modeled on Visual Basic. It is designed as a “lightweight” language with a fast interpreter for use in a wide variety of Microsoft environments.

You can use the built-in text editor for writing your script, but it is highly recommended that you use some kind of Interactive Development Environment (IDE) to allow for Syntax Highlighting, execution of the script and debugging. The scripts written for the stock reports were created using Notepad++ (http://www.notepad-plus-plus.org), but there are a plethora of IDE applications that you can use.
