ORDX Format

In general the ORDX format follows the formatting of Properties throughout a Job. Unlike an ORD file which is restricted to one Room the ORDX can have multiple Rooms. The ORDX can also define both Face and Interior Sectioning per Assembly which also can not be achieved via the ORD file.

We have included a sample ORDX file, which touches most areas of a Job, in the install. It is located in the \Support\Sample ORDX\ Folder.

As the format matures we'll be adding more documentation to this topic.

Working with Parts

There is a PartOverrides section in the ORDX. It uses the part path from the Object Tree to identify the part e.g. Cab.interior.AS@2. The standard 9 parameters are written out as well as other modified parameters which will be in a Parameter section.

<PartOverrides>

   <PartOverride>

      <Name>Cab.Case.LF</Name>

      <DZ>50</DZ>

   </PartOverride>

   <PartOverride>

      <Name>Cab.Case.TO</Name>

      <Y>760</Y>

      <DZ>20</DZ>

      <AX>-65</AX>

      <Parameter>

         <Name>GOAT</Name>

         <Type>M</Type>

         <Value>0.925</Value>

      </Parameter>

   </PartOverride>

   <PartOverride>

      <Name>Cab.Interior.AS@2</Name>

      <DZ>35</DZ>

   </PartOverride>

</PartOverrides>

 

There is also a Shape item in the PartOverrides section. In addition we also added Shape to the AdditionalParts section for shapes on manually added parts

<PartOverrides>

   <PartOverride>

      <Name>Cab.Interior.AS@2</Name>

      <Shape>

         <Axis>XY</Axis>

         <Template>

            ***

         </Template>

      </Shape>

   </PartOverride>

</PartOverrides>

 

 

The xml describes the shape as a series of entities (either lines or arcs). For a line you will have:


    <Begin>
        <x></x>
        <y></y>
    </Begin>
    <End>
        <x></x>
        <y></y>
    </End>
    

Everything is assumed to be Zero. So, if we were describing a line that started at (0,0) and went to (2,0) the only information that would be needed is:

     <entity id=”1” type=”line2d”>
         <End>
             <x>2</x>
         </End>
     <entity>

 

Here would be a simple triangle going from (0,0), (2,0), (1,1)

 

     <Shape>
         <Axis>XY</Axis>   XY is the default.  Other options would be XZ or YZ
               <template firstid=”1” unit=”mm”>
                    <entity id=”1” type=”line2d”>
                        <End>
                            <x>2</x>
                        </End>
                    </entity>
                    <entity id=”2” type=”line2d”>
                        <Begin>
                            <x>2</x>
                        </ Begin >
                        <End>
                            <x>1</x>
                            <y>1</y>
                        </End>
                    </entity>
                    <entity id=”2” type=”line2d”>
                        < Begin >
                            <x>1</x>
                            <y>1</y>
                        </ Begin >
                    </entity>
               </template>
     </Shape>

Note: There are also Arc entities.  It is probably best to output some cabinets / parts with various shapes to get a handle on the output.

Also any entity can have parameters. The format for an entity with parameters is:

     <entity id=”2” type=”line2d”>
         <Begin>
             <x>2</x>
         </ Begin >
         <End>
             <x>1</x>
             <y>1</y>
         </End>
         <parameters>
             <parameter name=”ABC”>
                 <value type=”4”>1</value>
             </parameter>
             <parameter name=”Test”>
                 <value type=”4”>1</value>
             </parameter>
         </parameters>
     </entity>
