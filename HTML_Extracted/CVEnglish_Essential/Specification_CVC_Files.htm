The *.CVC Files

The CVC database(s) consists of a list of Cabinets. These Cabinets can be built out of various materials; the material information is stored in the CxMaterials file. For each object library in CABINET VISION there is a corresponding .CVC file of the same name.

The following is the breakdown of the tables and their fields:

Accessories

This table stores general information for all Accessories.

Field

Description

AAssemblyID

A cross reference to the [Assembly].[AssemblyID] field.

Graphic

A Boolean value set to true if the accessory is visible, false if the accessory serves as an up charge.

ColumnPricing

A Boolean value set to true if the accessory has multiple price options.

UpCharge

A flat cost associated with the accessory.

UpChargePercent

The up charge percent this accessory is to add to a Cabinet or Job.

Assembly

A Boolean value set to true if the accessory has children, set false if the accessory is only one part.

Assembly

The catalog is a series of assemblies. The Assembly table has a list of all the assemblies.

Field

Description

A unique number to reference this assembly.

The name of the object.

A reference number to the main assembly if this object has been cloned. Zero, if the object is a main assembly.

The default height of the object. If the object is saved with the Preserve Size option selected it will come out of the library to the saved size. Otherwise, it will revert to the default height for this object type.

The default width of the object.

Depth

 

The default depth of the object. If the object is saved with the Preserve Size option selected it will come out of the library to the saved size. Otherwise, it will revert to the default depth for this object type.

Associations

The Associations table contains information that allows greater flexibility in specifying how stock doors are built and priced.

Description

A long integer value that contains the Door Style ID from the catalog’s Styles table.

A long integer value that contains the Door ID from the catalog’s Styles table BaseID, WallID or DrawerID fields and is the door ID from the door database specified in the catalog’s [Header].[DoorDatabase] field.

A long integer value that specifies the database ID of this associate. It can be the ID of another door, drawer, material or profile.

A double precision float that contains the dollar value of this association (may be zero).

A double precision value that contains the percent up-charge value of this association (may be zero).

A double precision value that contains the maximum allowable drawer height, only applies if this association is a drawer front (may be zero).

A Boolean value that specifies whether this association is available for use or not. Used primarily for internal maintenance.

A boolean value set true if the door catalog cost should be added to the UpCharge

LinePricing

This table stores the data that appears in the Stock Pricing section of a Catalog. This data can be viewed at either the Bid Center or through the Catalog Editor. It is basically a series of pricing equations that are evaluated in sequence to get the base, list, cost and selling price for each Cabinet in a Stock Catalog.

Field

Description

LineID

A unique number to identify the record

Order

The order in which the record will appear and be evaluated in the stock pricing table

Formula

The pricing equation for this line item.  Equation can consist of raw numbers, job totals, variables, previous line numbers and options

Reference

A short unique string to identify the purpose of the record

Description

A description of the record

System

A Boolean flag indicating that the record was created by the system.  These formulas can not be modified by either the manufacturer of the dealer

Locked

A Boolean flag indicating that the record was created by the manufacturer.  These items can only be modified from the catalog editor

ShowAtTotal

A Boolean flag indicating if this record Reference and Result of Formula should appear in the bid center as a price total.  Here the manufacturer or the dealer can decide what data is displayed

PerJob

A Boolean flag indicating in the record should be evaluated for each cabinet or only once per job.  As an example… If you had a 20 cabinet job and the formula was simply 100 and PerJob = $100 -- Not PerJob = $2,000 ($100 x 20 cabinets)

ShowAtCabinet

A Boolean flag indicating if this record should appear for an individual cabinet breakout in the bid center.

ShowAtAcc

A Boolean flag indicating if this record should appear for an individual accessory breakout in the bid center.
