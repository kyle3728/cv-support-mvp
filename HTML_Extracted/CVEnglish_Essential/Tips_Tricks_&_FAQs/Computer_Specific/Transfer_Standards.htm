Transferring Your Standards

NOTICE: This Topic assumes you are familiar with File Management in the Windows Operating System.

Follow these instructions to transfer files into Solid Essential 10 from previous versions or different computers.

The local hard drive is being referred to as "C" since that is the most common drive designation, however, it simply represents the drive where Solid has been installed. If Solid has been installed on a different drive on your computer, then that is the one you will use.

This topic describes the procedures for a standalone computer, but it will be the same if you are using a network server.

Step 1:

Open the Solid Backup Utility. From the Solid Splash Screen, click Utilities and then Backup Utility.

Step 2:

From the left side of the screen, check all the choices.

If you do not use the S2M Center, then do not check CNC Database or Posts.

From the Right side of the screen, check "Default.dat" and any new Catalogs (.cvc) that you have added or any that you have modified.

Click OK.

When the backup utility is finished running you must CLOSE SOLID.

Step 3:

This creates a folder that contains each of these selected files plus a zipped file called "backup.zip" which contains all the files in a compressed format. This new folder will be labeled with today’s date and time and will be placed within the Automatic Backups folder. This folder is located on your hard drive under the Planit\Solid_10_0 folder.

Now we need to transfer this newly created backup folder to the Automatic Backups folder within Solid Essential 10. If both versions of Solid are on the same computer, simply copy and paste this folder from the older Automatic Backups folder to the Solid 10_0 \ Automatic Backups folder.

If you are transferring these files from one computer to another, simply copy and paste this backup folder to a USB memory stick or whatever method of transfer you prefer, then copy and paste it to the Solid 10_0 \ Automatic Backups folder on the new computer.

At this point, make sure that Solid Essential 10 is closed.

Now that the new folder has been copied to the C: \ Planit \ Solid 10_0 \ Automatic Backups folder, follow these steps:

  1. On your computer, go to the Start button.

  2. Go to All Programs Solid Essential \ Restore Settings. The following screen will appear:

  1. The screen that pops up shows the available Backup folders on the left and the Files within each folder on the right.

  2. Make sure you have the folder you just transferred highlighted on the left.

  3. Now check which (or all) files you want to restore (transfer) on the right.

  4. Click OK

This will copy the selected files into the appropriate Database folder and run the appropriate update routine.  Once the update routine has finished running, Solid is now ready for use and will contain the updated information.

Moving Existing Jobs

If you want to also move your existing jobs, you can simply copy and paste all the existing job files (.cvj files) from one computer or folder location to another for use in Solid 10. These job files are usually contained within the C: \ Planit \ Solid 10.x \ Jobs folder, but they may be stored in any folder you have designated.

 A job that is edited in Solid 10 can no longer be used in an earlier version. Use a copy of that original job for safety. No update routine has to be run after copying these files.

Door Catalog

Now that the new Door catalog has been copied over, you will need to update the Door icons. Open Solid Essential 10, go to Door Catalog \ Setup and run Regenerate Door icons.

Usability of new files

While we make our best effort at making the older files compatible with the newer version of our software, we can’t guarantee all files will work exactly as they did in the older version due to the updates and additions to the new version. Existing jobs may not perform as expected. Also, the Construction Methods and User Created Standards will have to be reviewed to account for new or updated routines. Sometimes, older User Created Standards no longer work or are no longer needed. Please test the new information thoroughly before beginning production.
