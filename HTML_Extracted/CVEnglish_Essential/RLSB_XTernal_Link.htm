Transmit to Factory -

XTernal Link

This option allows passing of data to 3rd party or custom applications. You can have up to 5 unique Transmit to Factory Links:

LinkCommand LinkCommand2 LinkCommand3 LinkCommand4 LinkCommand5
LinkLabel LinkLabel2 LinkLabel3 LinkLabel4 LinkLabel5
LinkImage LinkImage2 LinkImage3 LinkIcon4 LinkImage5

This topic contains information about modifying the Windows™ Registry. Before you modify the Registry, make sure to back it up and make sure that you understand how to restore the Registry if a problem occurs. For information about how to back up, restore, and edit the Registry, click the following article number to view the article in the Microsoft™ Knowledge Base:

256986 Description of the Microsoft Windows Registry

An additional user defined button can be added to the sidebar in the Reports area. This button can be customized to open and populate Report.mdb, open and populate ORD Files, or to Open 3rd Party and/or Custom Applications. The feature can be activated by adding a LinkCommand string value to the registry that is a statement to be executed when the button is pressed. The LinkCommand string value should be added to the following Registry location:

HKEY_CURRENT_USER\Software\Cabinet Vision\Solid_11\Settings

It can contain one of the following variables: jobordfile, ordfile, user, reportdb, biddb. The variables must be within {} as normal Solid variables. If the ordfile variable is used, you will be prompted to save an ord file.

The button text can be set with a LinkLabel registry setting.

The image for the button can be set with a LinkImage registry setting. The image should be a 50px wide and 40px high .PNG file with a transparent background for the best results.

An example LinkCommand might be:

LinkCommand=notepad.exe "{jobordfile}"

This button, when clicked, would prompt you to save the Job File if it has not been and Saves an ORD file with the same name and location as the Job file and then open Notepad with the saved file...

notepad.exe "{ordfile}"

This button, when clicked, would prompt you to save an ORD file and then open Notepad with the saved file...

Another example might be:

LinkCommand="C:\Program Files\Microsoft Office\Office\msaccess.exe {reportdb}"

This opens a POPULATED report.mdb in MSAccess, without having to visit Report Center or Bid Center first.

Another example might be:

LinkCommand=notepad.exe "{user}"

This opens Windows Notepad with the Login User Name for Solid.

Another example might be:

LinkCommand="C:\Program Files\Microsoft Office\Office\msaccess.exe {biddb}"

This opens a POPULATED BidData.mdb in MSAccess.

 

There is another related Registry setting which can be entered here which will force CABINET VISION to populate Report.mdb upon entry into the S2M CENTER.

CNCPOPULATEREPORT

A value of 1 turns this feature on.
