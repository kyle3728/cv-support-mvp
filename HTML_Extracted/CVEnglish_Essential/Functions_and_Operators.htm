Functions and Operators

Supported Operators and Mathematical Functions Computation Operations

Operators may be binary or unary.  Binary operators have two operands such as in A * B.  A and B are the two operands in that expression.  A unary operator has only one operand such as in !B.  In that expression, the (!) is a unary operator (meaning Boolean Not), because it has only one operand (B).

(-) - Unary Minus

Example: -X

(-) - Binary Minus

Example: X - Y = Z

(+) - Unary Plus

Example: +X

(+) - Binary Plus

Example: X + Y = Z

(*) - Multiplication

Example: X  = X * 2Y

(/) - Division

Example: X = DX/2

(%) - Modulus

Example: X = Y % Z

(^) - Exponential

Example: X = Y ^ Z     (Raises Y to the power of Z)

 

Comparison Operators

Comparison Operators return a Boolean Value (True or False) depending on the comparison being made.  For example, the expression 1 == 2 would return False.

(!) - Logical NOT

Example: !a

(<) - Less Than

Example: X < Y

(>) - Greater Than

Example: Y > X

(<=) - Less Than or Equal To

Example: X <= Y

(>=) - Greater Than or Equal To

Example: Y >= X

(==) - Equivalence

Example     X == 1 (Must be used in Object Tree IF statements)

(!=) - Inequality

Example: X != Y

(&) - Logical And

Example; X & Y

(|) - Logical OR

Example: X | Y

 

Checking for Null Values or a Parameters Existence

(null) - Null

Example:   If DX = null Then

DoSomething

End If

Assignment Operators

An assignment operator sets or re-sets the value assigned to a variable.

Valid assignment operators in CABINET VISION are:

(=) - Equals

Example:  X = 2 + 2;  In this example, the variable (X) is assigned the value 2 + 2.

(:=) - Compute Now

Example:  X := 2 + 2;  In this example, the variable (X) is assigned the value 4.  This assignment operator varies from the (=) assignment operator in that with (:=):

  1. The expression is evaluated in the current state of the program, and

  2. The variable is assigned the computed value, replacing the prior value of that variable.

(+=) - Adjust Add

Example: DX += 2  (This adds 2 to the current DX value)

(-=) - Adjust Subtract

Example: DX -= 2  (This subtracts 2 from the current DX value)

(*=) - Adjust Multiply

Example: DX *= 2  (This multiplies the current DX value by 2)

(/=) - Adjust Divide

Example: DX /= 2  (This divides the current DX value by 2)

 

Deleting a Parameter (Via UCS)

(Delete) - Delete

Example: Delete DX (This will remove the DX Parameter from the Object)

 

The "This" Object

The "this" object is a substitute for what is in the For Each statement. The example below shows the difference in using it:

;Not Using the THIS Object

For Each DWR_OPEN|DOR_OPEN Part

If DWR_OPEN.DOR.S_DSLAB.DX > 5 Then

Do Something

End if

If DOR_OPEN.DOR.S_DSLAB.DX > 5 Then

Do The Same Something As Above

End if

The code above when using the "this" object could be replaced with:

;Using the THIS Object

For Each DWR_OPEN|DOR_OPEN Part

If this.DOR.S_DSLAB.DX > 5 Then

Do It All

End if

 

Define Value types in a UCS

You can define the following Value types via a UCS.

Currency <crncy>

Measurement <meas>

Degrees <deg>

Integer <int>

Boolean <bool>

Decimal <dec>

Text <text>

You can also build strings of Text in CABINET VISION from the results of existing Parameters or AutoText Values. Such as the three examples below:

NAME = w{DX}h{DY}d{DZ}

PARTMAT = {material}

COMMENT = {name} ({job.name})

Style <style>

There are three possible settings for the <style> value:

0 = Value

Values act as previous Parameter definitions, only displayed in the Object Tree and hard set by the UCS.

1 = Attribute

Attributes act as triggers that can be user prompted from the sidebar for the Object or via Job/Room Parameters tabs. Use <desc> (below) to set the prompt description.

2  = Note

Notes are a user defined Note which will be displayed on the Notes pages for the Object and available as CAD text look-ups. The text entered for Notes are default values which can be edited from the prompt. Use <desc> (below) to set the prompt description.

Description <desc>

Description allows you to display a user prompt for Attribute and Note Parameters.

 

Below is a sample UCS showing how to format utilizing these functions:

;Assigning Value Types in a UCS

For Each CAB Assembly

 

If U_DWStrip = null then                                              ;Checks to see if the U_DWStrip already exists on the Cabinet

     U_DWStrip<bool> = 0                                            ;Sets the Parameter Type to Boolean (True/False) with a default of False

     U_DWStrip<style> = 1                                           ;Sets the Parameter Style to Attribute

     U_DWStrip<desc> = 'Add Dishwasher Strip'         ;Adds a user prompt at the sidebar for the Cabinet  

End If

 

If Cab.U_DWStrip = 1 Then                                         ;Checks to see if the User said Yes to Add Dishwasher Strip Question

  Dim BOARD as New Part

  BOARD.X<meas> = 0                                                ;meas = Measurement

  BOARD.Y<meas> = Cab.DY                                     ;meas = Measurement

  BOARD.Z<meas> = 0                                                ;meas = Measurement

  BOARD.DX<meas> = 1.75                                        ;meas = Measurement

  BOARD.DY<meas> = 24                                           ;meas = Measurement

  BOARD.AX<deg> = 0                                                ;deg  = Degrees

  BOARD.AY<deg> = 0                                                ;deg  = Degrees

  BOARD.AZ<deg> = -90                                             ;deg  = Degrees

  BOARD.QTY<int> = 3                                                ;int  = Integer

  BOARD.WEIGHT<dec> = 23.25                                ;dec  = Decimal

 

  NOTE<text> = 'Extra Board for Mounting Dishwasher Included'        ;defines the text contained in NOTE

  NOTE<style> = 2                                                                                ;sets the Parameter NOTE as a Note on the CAB in For Each

End If

 

You can also define a list of values for a user to select from. See the Parameters topic for more information.

It is possible to process variables in-line. The variables should be within {} braces. The following demonstrates the use of a variable in  "while...end" and "if...then" statements.

 

For Each AS Part

 

_COUNT = 1

_NAME<text> = 'AS'

 

While CAB.INTERIOR.{_NAME}@{_COUNT}.DX > 0 Do

     If CAB.INTERIOR.{_NAME}@{_COUNT}.ID == ID Then

          _FOUND := ID

          Exit

     End If

     _COUNT += 1

End While
