User Created Standards

This Utility allows you to Create and Modify User Created Standards.

Your UCS' will be listed in this window. Following are the available columns in the UCS list.

Scripts Tab

Type Column

The first column denotes the UCS Type. There are two possibilities:

  •  - Denotes a classic UCS Macro (UCS:M)
  •  - Denotes a JavaScript UCS (UCS:JS)

Order Column

Denotes the priority in which the User Created Standards will be processed. Use the Up and Down buttons on a highlighted UCS to change it's priority.

Name Column

This is the Name of the User Created Standard. A second click in the field will allow you to rename the UCS.

Description

The Description is pulled from the first remark line in the User Created Standard.

Part Column

The Primary Part that was selected when the User Created Standard was created.

Pre-Build

Lets you set whether the UCS is set to Apply Pre-Build or not.

Enabled Column

This will determine if your UCS is Enabled or not.

Libraries Tab

The Libraries tab is a place to use to add Libraries of functions for UCS:JSC (JavaScript UCSs) to utilize.

In a traditional UCS (UCS:M), code is often duplicated in each UCS:M which can cause code bloat and difficulty in ensuring propagating any modifications to the code made it to each instance of the replicated code. With the added power and flexibility of JavaScript, it is even more likely there will be a use for commonly used functions. To fulfill this need, in a UCS:JS, Libraries (Libs) can be used to create groups of functions that can be shared between scripts without the need for duplication.

A library will typically contain functions with related purpose, for example a math or geometry library. In code the libraries are implemented as a JavaScript class with static methods (functions).

class ClassName {
  method_1() { ... }
  method_2() { ... }
  method_3() { ... }
}

The name given to the JS Lib in CV will be used as the class name using the format _libraryname. (the library name in lower case, preceded by an underscore).

Example:

A library can be created in CV called “Math”


In the code, methods for area and volume could be defined.

area(width, length)
{
  return width * length;
}
volume(width, length, depth)
{
  return width * length * depth;
}

In a UCS, the library can be used as follows

var width = _this.GetParameterValue('DX');
var length = _this.GetParameterValue('DY');
var depth = _this.GetParameterValue('DZ');
_this.SetParameter('MYAREA', _math.area(width, length));
_this.SetParameter('MYVOLUME', _math.volume(width, length, depth));

Public Variables

The Public Variables area in the sidebar will list any Public Variables that have been defined in the selected UCS:M. See the Public Variables topic for more information.

Public Variables can not be defined within UCS:JS so must be manually added here. When a UCS:JS is selected you will see the Add Public Variable icon  become available. Simply define your Public Variable here.

Ribbonbar

Following are the ribbonbar options on the User Created Standards main screen:

 Return 

Closes the User Created Standards window.

New

Copy

This option creates an exact copy of the selected UCS.

Delete

This option will delete the currently selected UCS.

 Set Columns

Wizard

Code

 Convert to JS

This option will convert the currently selected UCS Macro (UCS:M) to a JavaScript UCS (UCS:JS). A copy of the existing UCS:M will be kept in the background for use y the Restore UCS commend if it is ever necessary.

 Restore UCS

This option will restore a UCS Macro (UCS:M) from one previously converted to a JavaScript UCS (UCS:JS).

 When using the Restore UCS option the existing UCS:JS will be lost.

 Add Apply Conditions

There is no concept of a "For Each" statement with UCS:JS so when a UCS:JS is selected this option allows you to define the Objects that the UCS:JS needs to act on.

 You can use wildcards here such as * or ? to represent single characters. If using * it is highly recommended to use the OBJECT parameter to filter the number of objects the UCS is applied to.

Other Buttons

 The Up button will move the selected UCS(s) Up in Order of priority in the list.

 The Down button will move the selected UCS(s) Down in Order of priority in the list.

 The units button will toggle the unit of measure between Inches and MM.

 The precision button will allow you to set the measurement precision for the selected unit of measure.

 The Close button will exit out of User Created Standards.
