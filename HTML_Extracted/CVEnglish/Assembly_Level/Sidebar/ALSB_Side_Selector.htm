Side Selector Section View - Section Face

This Side Selector interface allows you to select the Side of the Assembly you would like to modify in the Section Editor. In the image below the Assembly was edited in an Ortho View so that the Top and Front of the Assembly were setup to have Faces. When Faces are defined on two different planes it activates the plane selection drop down at the Top. In the View below we have the End View selected so we can define the Top (Priority 1), the Front (Priority 2), the Back, and the Bottom of the Assembly.

Face Priority Explanation

The face priority is the order in which the interior of the Assembly will be sectioned. It is done in the order 1, 2, 3, etc. If you put in a Partition on Face priority 1 it will cut all the way through the Assembly. If you then put a Partition on Face priority 2, the Partition will stop where it meets Partition 1. If you change the priorities around so 2 becomes 1, then the old 2 will now go all the way through the Assembly and the old 1 will stop where it meets the other Partition.

Side Selector with a Standard Assembly

When you first enter the Section Editor with a Standard (Non-Shaped) Assembly you will be presented with the following sidebar view.

You can select any of the four Sides presented in the Side Selector. By selecting either of the Ends represented by the Side selector you will receive the following options:

  • Side Types - Here you can select whether the selected Side in the Side Selector is defined as a Face or an End. If you select Face you can then Section it just like a standard Assemblies Face.

  • End - If End is selected in Side Types, as is the case in the image above, you can define the Ends Type. There are eleven options here:

    1. Automatic - When Automatic is selected as the End Type you give Solid control to determine what the End type is based on it’s condition. Basically this means it follows the logic of Unfinished End when against a Wall or other Assembly or Finished end when the End is exposed.

    2. Unfinished - Changes the End type to an Unfinished End following its Properties as defined in whichever Construction Method that is active. This is an Override to CABINET VISION’s "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    3. Finished - Changes the End type to a Finished End following its Properties as defined in whichever Construction Method that is active. This is an Override to CABINET VISION’s "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    4. Applied Finished End - Changes the End type to an Applied Finished End following its Properties as defined in whichever Construction Method that is active. This is an Override to CABINET VISION’s "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    5. Applied Door - Changes the End type to an Applied Door Finished End following its Properties as defined in whichever Construction Method that is active, utilizing the selected Door Style for the Classes assigned End Panel. This is an Override to CABINET VISION’s "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    6. Panelized - Changes the End type to a Panelized Finished End following its Properties as defined in whichever Construction Method that is active, utilizing the selected Door Style for the Classes assigned End Panel. This is an Override to CABINET VISION’s "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    7. Extended – Changes the End type to an Extended Unfinished End following its Properties as defined in whichever Construction Method that is active. This is an Override to CABINET VISION’s "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    8. Return - Changes the End type to a Return Unfinished End following its Properties as defined in whichever Construction Method that is active. This is an Override to CABINET VISION’s "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    9. Filler - Changes the End Type to Filler. This is an Override to CABINET VISION’s "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    10. Skin - Changes the End Type to Skin. This is an Override to CABINET VISION’s "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    11. No End - Changes the End Type to No End. This is an Override to CABINET VISION’s "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

By selecting the Back represented by the Side selector you will receive the following options:

  • Side Types - Here you can select whether the selected Side in the Side Selector is defined as a Face or a Back. If you select Face you can then Section it just like a standard Assemblies Face.

  • Back - If Back is selected in Side Types, as is the case in the image above, you can define the Backs Type. There are five options here:

    1. Automatic - When Automatic is selected as the Back Type you give CABINET VISION control to determine what the Back type is based on it’s condition. Basically this means it follows the logic of Unfinished Back when against a Standard Wall or Finished Back or No Back when against a Peninsula Wall, depending on its Back Type Property.

    2. Unfinished - Changes the Back type to an Unfinished Back following its Properties as defined in whichever Construction Method that is active. This is an Override to CABINET VISION’s "Automatic" Back Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    3. Finished - Changes the Back type to a Finished Back following its Properties as defined in whichever Construction Method that is active. This is an Override to CABINET VISION’s "Automatic" Back Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    4. Clipped Back Support - Changes the Back Type to a Clipped Back Support following its Properties as defined in whichever Construction Method that is active. This is an Override to CABINET VISION’s "Automatic" Back Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    5. No Back - Changes the Back Type to No Back. This is an Override to CABINET VISION’s "Automatic" Back Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.
