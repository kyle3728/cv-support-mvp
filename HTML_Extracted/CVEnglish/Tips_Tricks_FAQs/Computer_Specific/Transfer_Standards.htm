Transferring Your Standards

 This Topic assumes you are familiar with File Management in the Windows Operating System.

Follow these instructions to transfer files into CABINET VISION Version 2024 from previous versions or different computers.

The local hard drive is being referred to as “C” since that is the most common drive designation, however, it simply represents the drive where CABINET VISION has been installed. If CABINET VISION has been installed on a different drive on your computer, then that is the one you will use.

This topic describes the procedures for a standalone computer, but it will be the same if you are using a network server.

Step 1:

Open the Backup Utility. From the CABINET VISION Main Screen, click Utilities and then Backup Utility.

Step 2:

At a minimum you should "Include" what is in the "Include" area in the illustration below.

Click Save.

When the backup utility is finished running you must close CABINET VISION.

Step 3:

This creates a folder that contains each of these selected files plus a zipped file called “backup.zip” which contains all the files in a compressed format. This new folder will be labeled with today’s date and time and will be placed within the Automatic Backups folder. This folder is located on your hard drive under the Planit or the Cabinet Vision \Solid 8.0, 9.0, 10.0, etc... folder. If you have 3.5 or older, it will be in the Cabinet Vision \ Solid folder.

Now we need to transfer this newly created backup folder to the Automatic Backups folder within CABINET VISION Version 2024. If both versions of CABINET VISION are on the same computer, simply copy and paste this folder from the older Automatic Backups folder to the following folder:

C:\ProgramData\Hexagon\CABINET VISION\Common 2024\Automatic Backups\

If you are transferring these files from one computer to another, simply copy and paste this backup folder to a USB memory stick or whatever method of transfer you prefer, then copy and paste it to the folder above on the new computer.

At this point, make sure that CABINET VISION Version 2024 is closed.

Now that the new folder has been copied to the C:\ProgramData\Hexagon\CABINET VISION\Common 2024\Automatic Backups folder, follow these steps:

  1. In Windows 10 or 11 click the Start button or press then select All Apps and scroll to the CABINET VISION 2024 Group and run Restore CV 2024 Settings.

  1. The following screen will appear:

  1. The screen that pops up shows the available Backup folders on the left and the Files within each folder on the right.
  2. Make sure you have the folder you just transferred highlighted on the left.
  3. Now check which (or all) files you want to restore (transfer) on the right.
  4. Click OK

This will copy the selected files into the appropriate Database folder and run the appropriate update routine. Once the update routine has finished running, CABINET VISION is now ready for use and will contain the updated information.

Moving Existing Jobs

If you want to also move your existing jobs, you can simply copy and paste all the existing job files (.cvj files) from one computer or folder location to another for use in Version 2024. These job files are usually contained within the C:\Users\Public\Documents\CABINET VISION\CV 2024 folder, but they may be stored in any folder you have designated.

 A job that is edited in Version 2024 can no longer be used in an earlier version. Use a copy of that original job for safety. No update routine has to be run after copying these files.

Door Manager

Now that a new Door catalog has been copied over, you will need to update the Door icons. Open CABINET VISION 2024, go to Door Manager \ Utilities and run Generate Icons.

S2M Center

If you use the S2M CENTER, you will have to update these items:

  1. Reselect the Primary and Secondary machines you use for Optimizing.
  2. Create a new Output Directory (path) for your output code folder.
  3. Reselect the path to your Post for each machine you use.
  4. Re-enter your Post code if applicable

Usability of new files

While we make our best effort at making the older files compatible with the newer version of our software, we can’t guarantee all files will work exactly as they did in the older version due to the updates and additions to the new version. Existing jobs may not perform as expected. Also, the Construction Methods and User Created Standards will have to be reviewed to account for new or updated routines. Sometimes, older User Created Standards no longer work or are no longer needed. Please test the new information thoroughly before beginning production.
