Adding A List To A Label

There are times that a small list of information must be added to a label. You can do this using CV's built in concat functionality.

What does concat mean?

When you concatenate (or concat for shorthand) a string, it means you are taking a variable number of strings, like:

"Hello"

"World"

And putting them together into a single string:

"Hello World"

As an example, let's create the following view in Cabinet Vision:

DoorLabels

SELECT Doors.[Cabinet ID], Doors.[Label] + ' - ' + Doors.[Type] as DoorLabel

FROM Doors

For my database, that creates the following view in the Report.mdb:

We can see that, in CABINET VISION, I have 4 cabinets that have 3 doors each (I basically created a cabinet and copied it 3 times).  Since labels can't create lists, we need to use this feature to get what we want, which would look like this:

As you can now see, all items that have a related 'Cabinet ID' (meaning a 'Cabinet ID' that is the same) have been combined, or rather concatenated, together into a single string that we can now place on our label.

Step by Step:

  1. Create a new report in CABINET VISION.

  2. Right click on the Report and select Properties.

  3. Create an 'Import'.

    1. Name: DoorLabels

    2. Type: Link

    3. SQL: SELECT Doors.[Cabinet ID], Doors.[Label] + ' - ' + Doors.[Type] as DoorLabel FROM Doors

  4. Create a new table.

    1. Make this a cabinets table.

    2. On the comment list, select the following:

      1. Primary Field: Cabinet ID

      2. Foreign Table: DoorLabels

      3. Foreign Field: Cabinet ID

      4. Display Field: DoorLabels

Now, when you run your report, you will have a new variable, that is in a sub-folder of the table's folder, with your door labels, that allow you to place this list on the label.
