Formula for the DY (Length)

Look at your cabinet in 3D. Look at it from a front ortho view and click on the BOARD.  Notice where the reference point is located.  When writing code that sizes a part it is important to know where the reference point is.  Because when the part changes size it could also change position, if your not careful how you orient the part.

There are several formulas that will work for the DY of the BOARD.  We could say the width of the cabinet minus the thickness of the two ends.  But we already know to stay away from the ends if we can.  We could say the width of the cabinet minus 1 1/2", but if the thickness of the ends changed this code wouldn’t work very well.  Like I said earlier the easiest thing to do is find something in the cabinet that all ready has the value you need and that will change in the way you want the parameter you are working on to change.

If you look at the cabinet in 3D you can see that the Deck, the Top Stretchers, and the Nailers are all butted between the sides of the cabinet, the same as this BOARD.  If the thickness of ether end changes the Deck, the Top Stretchers, and the Nailers will all change size in relation to the thickness of the ends.  Also, if the cabinet is stretched so that it is wider, these parts will grow.  So we could use the DY of any of these parts as the DY of our board.  We will use the DY of the deck for the DY of our BOARD.  So our formula is: the length of the deck.  So now that we have our formula.  We need to trace it through the object tree to get our code.

  1. Click on one of the ortho views.

  2. Click on "Object Tree".

  3. Open the object tree until you can see the Deck.

  4. Click on the DY parameter.  Trace it through the tree and it is Cab.Case.De.Dy write this down, this is the code we need.

  5. Click on the BOARD and then click on the length (the DY) then click on "Edit".

  6. Change from a "Value" to an "Equation".

  7. On the THEN line type in the code we came up with Cab.Case.De.Dy.

  8. Click on the "Test" button and it should say "Condition returned: True. Equation returned: 34 1/2 or whatever value you were expecting. Click on OK.

  9. Look at your cabinet in 3D does it look OK?  If not you typed something wrong.  See "Troubleshooting Error Messages".

Try changing the size of the cabinet the width, the height and the depth.  The BOARD we added should move and size correctly.  When done put it back to the original size.  You might want to save the cabinet back into your library so you can do the second example exercise later.
