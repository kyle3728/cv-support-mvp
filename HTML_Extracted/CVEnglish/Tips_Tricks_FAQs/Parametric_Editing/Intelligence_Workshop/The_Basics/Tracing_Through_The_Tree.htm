Tracing Through The Tree

If I asked you what the height of a Cabinet was you might say 34 1/2". This is correct as far as it goes. But to tell the computer what height the Cabinet is, is done a little differently. To tell the computer how high the Cabinet is, you would say CAB.DY.

If you wanted to subtract the thickness of the Deck from the height of the Cabinet you would say 34 1/2 – 3/4 = 33 3/4". To tell the computer the height of the Cabinet minus the thickness of the deck, we would say CAB.DY - CAB.CASE.DE.DZ. To come up with this information you need to trace it through the Object Tree. See the following picture. Let's take the height of the Cabinet minus the thickness of the Deck and trace it through the tree. First we want the height of the Cabinet.

Open the Object Tree (see Object Tree "Assembly Level") and click on the Cabinet to highlight it. Then look down at the lower pane and the parameter for the height of the Cabinet is "DY". So the height of the Cabinet is CAB.DY.  The very top level of the Object Tree (when you are dealing with an Assembly) is called CAB. It doesn't matter what the name of the Assembly is, for formulas it is called CAB.

Next get the thickness of the Deck. Open the Object Tree up even more until you find the Deck. Starting at the top (CAB) and working your way down, the thickness of the Deck is CAB.CASE.DE.DZ.

Where are we coming up with those periods? Well every time the tree branches we add a "period". Notice that in the case of the Deck it starts at the top (CAB) and then it branches at Case so you add a period between the word CAB and the word Case. Next it branches at the Deck so you add a period between Case and DE and you have CAB.CASE.DE. From the Deck you move to the parameters in the bottom pane. Any time you move from the top pane to the bottom pane you add a period so you must place a period between DE and the parameter DZ (DZ = thickness which is what you are looking for) and you have CAB.CASE.DE.DZ, which is the thickness of the Deck. So the height of the Cabinet minus the thickness of the deck is equal to CAB.DY-CAB.CASE.DE.DZ.

Say you wanted to know the width of the Adjustable Shelf minus the thickness of the Adjustable Shelf Strip. First you would open up the Object Tree until you found the Adjustable Shelf. See the following picture. If you trace the first part, the width of the Adjustable Shelf through the tree you come up with CAB.INTERIOR.AS.DX. If you trace the second part through the tree you come up with CAB.INTERIOR.AS.SS.DZ. Putting these two together you come up with CAB.INTERIOR.AS.DX – CAB.INTERIOR.AS.SS.DZ.

So how does CAB.INTERIOR.AS.DX – CAB.INTERIOR.AS.SS.DZ work?  Well, it is like a road map for CABINET VISION. The first part CAB.INTERIOR.AS.DX leads CABINET VISION to the DX parameter of the "AS" (Adjustable Shelf), which in this case has a value of 22 23/32 inches. The second part CAB.INTERIOR.AS.SS.DZ leads CABINET VISION to the DZ parameter of the "SS" (Adjustable Shelf Strip), which has a value of 3/4 inches. The whole thing basically tells CABINET VISION 22 23/32 – 3/4 = 21 31/32.
