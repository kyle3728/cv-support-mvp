The Object Tree

The Object Tree gives you access to the parameters of the objects making up your Job/Room/Assembly/Part. It is called the Object Tree because it branches like a tree.

Object Tree (Room level)

The following picture shows a typical Object Tree from the Room Level. To access the Object Tree from the Room Level click the Object Tree tab  at the bottom of the sidebar.

The Object Tree represents the hierarchy (or chain of command) of a Job or an Assembly. Each item of a Job occupies a particular place on the "Object Tree" and its branching structure. If you were to view the Object Tree at the Assembly Level then the same tree structure applies, but only to the one Assembly.

Object Tree (Assembly level)

To see the Object Tree for a single object, say a Cabinet. Right click on the Cabinet and select "Edit". Then click the Object Tree tab  at the bottom of the sidebar.

Notice the nine basic parameters in the lower pane of the above screen. I know it has a couple extra parameters, don't worry about them for now. These parameters are for the "Finished Left End" (or LF) since it is the object highlighted.

The X (Left/Right) = 3/4" so the reference point of the end is 3/4" from the reference point of the Cabinet (3/4" from the left end).

The Y (Up/Down) = 4" so the reference point of the end is 4" up from the reference point of the Cabinet (4" up from the bottom).

The Z (Front/Back) = 0" so the reference pint of the end is lined up with the reference point of the Cabinet (0" from the back).

The reference point for the Cabinet is in its left, bottom, and back corner, and the reference point for the Part is in its right, bottom, and back corner.

The DX = 24 so the width of this end is 24".

The DY = 30 1/2 so the length of this end is 30 1/2".

The DZ = 3/4 so the thickness of this end is 3/4".

The AX = 0 so this Part has not been rotated on the X-axis.

The AY = -90 so this Part has been rotated on the Y-axis (clockwise) to face left.

The AZ = 0 so this Part has not been rotated on the Z-axis.
