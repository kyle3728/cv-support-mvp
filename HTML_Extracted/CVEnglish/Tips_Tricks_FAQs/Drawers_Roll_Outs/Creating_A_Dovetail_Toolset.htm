Creating A Dovetail Toolset

From the CV opening screen go to the ribbon option Main | Tool.

Click the "Create New Tool" tool in the toolbar.

Give your Dovetail Tool Set a name and select Dovetail Tool Set as the type.

Next select your new Dovetail Tool Set from the Tool list and fill in the Properties for your Dovetail Tool Set.

Dovetail Tool Set Properties

General

  • Machine - Select the Machine to assign this Tool to. Use the Any selection if this Tool is available for all Machines.

  • Name - Enter a Name to identify the Tool. The name you give is for your reference only. In some posts the tool name will appear in a comment in the g-code next to the tool change.

  • Tool Number - Critical for all tools except drills in a gang drill head. This is the ID that Intelli-CAM will use to match the Tool to the Machine in the Machine Catalog to the actual Tool on the Machine. For drills used in a gang drill head this value is ignored.

  • Height Offset ID - This is typically the same value as the tool number or the tool number + 100. This number refers to a preset value on the controller that defines the distance from the tip of the tool to the top of the spoil board or top of the Material.  This is a value set by the CNC operator In mathematics and in programming and computer applications, a symbol or other character indicating an operation that acts on one or more elements. You can use the following four operators in standard calculations: / divide * multiply - subtract + add.  It references the different heights of the tools above the work surface (In some posts this value is ignored).

  • Tool ID - This is a unique ID Number assigned by the S2M CENTER. This value can be called from within your design package to assign a specific Tool to a specific Operation. For Example, within CABINET VISION, in a User Created Standard you can use the Parameter ToolID equal to the Tools Tool ID number to make an assignment.

Tool Size

Items preceded by a letter designation reference the area noted with that letter in the preview area.

  • Diameter - The S2M CENTER will use this size to match tools to operations.

  • Work Depth - This field is used to Display the Graphic for tools. It holds no other importance.

  • (H) Pocket Width

  • (H) Pocket Tool

By default the Pocket created is an exact match to the Tool Defined (A, B, C, D, E). This option allows you to make a Pocket the is Wider than the Tool in order to perform more decorative "wide" dovetails.

  • (L) Tail Width Extension

This option allows you to extend the Width of the pocket to adjust fit.

 

Nesting

  • Nested Part Clearance - Establishes the Part Clearance necessary on the edges of the Part where the Dovetail Operation is performed.

Movement

  • Rotate Clockwise - This field in addition to the Material field "Climb Cut" will determine the Tool path direction.  Think of the router bit as a wheel.  If the path the wheel is following is the same direction, as a wheel would follow if rotated that direction, it is a "Climb Cut" otherwise, it is a "Conventional Cut".  An inside cut will always be in the opposite direction of an outside cut.  A "Conventional Cut" will always be in the opposite direction of a "Climb Cut".  A simple rule is: If the Material is set for a "Climb Cut" an outside pass would be in the same direction of Tool rotation.  Here is a chart to keep straight all the options: Remember the analogy to the wheel that will help keep the matter, including inside, outside, etc... straight.

Feed Rate

  • Feed Rate - This determines how fast the Machine will try to move the Tool through the Material.  This is often a difficult field to find values for since we lack a standard for feed rate.  The value represents Inches per minute, or MM per Minute. This can be even further confusing since some posts expect a unit but then convert the unit in the post.  For example the MultiCAM post expects Inches Per Minute but converts the final output to Inches per Second.  In addition the material will have an effect on the final feed rate.  When testing take into account the % feed rate property in the Material Catalog.

  • Descent Rate - This will have the same unit of measurement as feed rate.

  • Spindle Speed - This is the speed at which the tool will spin.  The unit is almost always RPM  Rotations per minute.  Setting a value of 0 here will usually cause an error in the G-Code.  A valid range would be about 4000 to 24000.  It would depend on the Machine and Tool's recommended specifications.  Some Machines will ignore this number and use a preset number for a Tool.
