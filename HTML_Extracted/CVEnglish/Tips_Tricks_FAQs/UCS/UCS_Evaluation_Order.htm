UCS Evaluation Order

This topic is a look into the way that CABINET VISION generates Assemblies and the order that it evaluates parameters. Following are the basic steps that CABINET VISION goes through to generate an Assembly:

  1. The Assembly is generated according to the Construction Method and creates the Connection specified between parts.

  2. User Created Standards are processed for that Assembly, it’s Parts, and the Connections. See below for details about the way that User Created Standards are processed for an Assembly.

  3. The parts are generated.

  4. Relocate the parts.

  5. Connector creates Operations.

    1. User Created Standards are processed for connection generated objects.

    2. Build the connection generated objects

    3. Relocate the connection generated objects

UCS Evaluation For a Assembly

When User Created Standards are evaluated for an Assembly, they are evaluated in the order that they (User Created Standards) occur in the list of User Created Standards. UCS #1 is evaluated for the current Assembly, then UCS #2 is evaluated for the current Assembly, etc.
