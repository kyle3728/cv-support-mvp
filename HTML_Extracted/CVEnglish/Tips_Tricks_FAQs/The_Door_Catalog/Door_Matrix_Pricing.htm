Door Matrix Pricing

Matrix Pricing was developed for the UK and Australia, this method for pricing Doors is performed with a size range per Material. Matrix pricing was added to facilitate this method. It is typically only enabled overseas, but can be turned on by adding MatrixPricing=1 to the Cabinet Vision\Solid_10_0\Catalogs\Door branch in the registry.

Editing the Windows Registry is only recommended for those familiar with the Windows Registry. Always make a backup of the Registry prior to editing. One wrong entry to could make it impossible to Open Cabinet Vision or even Windows. Be very careful!

The door companies would price all doors from 1x1 to 15x10 as a 15x10 door. All Doors from  1x10.125 to 15x20 as 15x20 door.

At this point a 14 x 18 Door would be Size F,  a 30 x 36 Door would be size S.

The next step in the Door Matrix pricing setup is to set prices for A, B, C etc.

The Door Material would determine a different set of prices even though the sizes were fixed. In other words Size F might be $34 dollars for Ash and Oak and $47 dollars for Maple. You would setup two prices for this Matrix, one called Ash/Oak the other called Premium (anything descriptive).

You can setup as many different Size Matrices as needed for the different Door manufacturers then can set pricing in the Size Matrices as needed.

At the Door Material tab you can then select a Matrix price for the Door (Ash/Oak).

Although users using a Price Matrix, typically, only price with the Matrixany other Door pricing you have setup will be added to the Matrix Price. In other words if your size F for a Oak door is $34 the final price for the Door will be more if you have a $/door or $/SqFt for the Door, Materials or Profiles.
