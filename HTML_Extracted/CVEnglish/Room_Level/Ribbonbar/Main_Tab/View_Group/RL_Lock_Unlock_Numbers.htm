/ Lock/Unlock Assembly Numbers

Normal Assembly numbering begins with Assembly #1. Which, is the first Assembly to appear in the Job. However, if an Assembly is added to the Room and it is added to a Wall with a lower number, it will renumber the Assemblies within the Job. Typically, Base Assemblies are numbered before Upper Assemblies. Lower numbers are on the left of a Wall, and the Upper Assemblies continue the numbering sequence that the last Base Assembly was using. If Wall #1 had 5 Assemblies, then Wall #2 would start with Assembly #6, and running left to right, Base then Upper Assemblies, assign numbers to those Assemblies. The next Room will start where the previous Room left off. Deleting or adding Assemblies to an earlier Wall or Room results in renumbering of all subsequent Assemblies, which can sometimes present problems. The lock Assembly Numbers feature prevents this potential problem.

CABINET VISION allows locking Assembly numbers that are program generated, or user selected, as well as assigning and locking a unique number to an Assembly.

Once locked, the numbers on the Assemblies will not change regardless of where other Assemblies are added or deleted within the Room. If Assembly numbers are locked, and an Assembly is added to that Room, it will receive the next higher number in sequence, assuming the number is not used elsewhere, as in another Room. Assemblies with unlocked numbers all start numbering from the lower left side of Wall #1 in Room #1 and proceed left to right, then left to right for Upper Assemblies on the same Wall, and then to the next Wall following the same numbering sequence. If an Assembly is added to the Base Assemblies on Wall #1, Room #1, ALL Assemblies renumber in all subsequent Rooms. This is prevented if Assembly Numbers are locked.

To lock a Job access the Utility/Lock Assembly Numbers as shown below. To unlock, access the same utility, and select the Unlock feature.

Remember, this is a Job Level feature, therefore, all Rooms are locked.

Assigning a unique number to specific Assembly

An Assembly can have a unique number assigned to it and locked. The only requirement is that the number is not previously used in the Job. An Assembly number can only exist one time in any Job. To lock a number to a specific Assembly, right click on the Assembly, and select the "Properties" option. The "General" tab allows assigning a number and locking that number by clicking on the box next to the number. If you wish to assign a unique number, providing it is not previously used, you can do so by replacing the number in the box with the unique number then checking the box.
