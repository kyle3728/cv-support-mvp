ORD File Format

Order Entry (ORD) Version 4 File Format

Standard, Extended, and Jobsite Companion

The Order Entry (ORD) file format is an ASCII format used for transferring CABINET VISION Cabinet data in a version and configuration independent form. The ORD file may contain Cabinets from multiple Catalogs with multiple door styles and options.

The ORD file has a hierarchical structure consisting of a [Header] section followed by [Catalog] and [Cabinet] section pairs. The [Header] section is used to set the CABINET VISION Job defaults while the [Catalog] and corresponding [Cabinet] section are used to set the Door Styles and options for a list of Cabinets. Multiple Catalogs, Door Styles and options are specified by including multiple [Catalog] and [Cabinet] section pairs in the ORD file.

For example, if all the Cabinets in a CABINET VISION Job had the same Catalog and options, the ORD file would only have one [Catalog] section followed by one [Cabinet] section. If there were some Cabinets from a different Catalog, there would be secondary [Catalog] and [Cabinet] sections. This scheme may be repeated as many times as necessary to list all the Cabinets in a CABINET VISION Job with differing Catalogs, door styles and options.

Certain characters in Comment fields can cause problems and should be avoided. This includes but is not limited to the following: # ? = | ;

Items in red represent the Extended ORD File Format

An Extended Format allows for the creation of Walls, Floors and the placing of Moldings and Cabinets on the Walls. The created objects are visible in the Job as if they had been placed in Layout whereas in the Standard ORD format, the Cabinets are added to an Order Entry List and are not visible in the Layout views. The Walls section should precede the Moldings and Cabinets sections as the items in these sections are attached to a Wall which needs to be defined first.

The Materials, Door Styles and Construction set here do not affect any of the Cabinets listed in the ORD file as they take all their configurations from their corresponding [Catalog] sections. Walls Section (Extended Format)

The entries in this section are used to define Walls. The parameters reflect the Wall’s location and direction as well as its Length, Height and Thickness. If the start or end point of a Wall is the same as an existing Wall, the Walls will be joined at that point.

If this section is present, all Cabinets will be added to a Wall instead of being added to the Order Entry list and will thus be visible in the Layout view (Floor Plan, Elevation and Perspective). The optional positional parameters for the Cabinets should be filled out to place the Cabinets correctly on the Wall.

To maintain a consistent Wall numbering sequence, the Walls should be defined in a clockwise direction. This will ensure that the Walls are numbered in the order in which they are defined.

Moldings Section (Extended Format)

The entries in this section are used to define moldings attached to a wall. The Molding type may be one of the following predefined types, "Crown", "LightRail", "Scribe", "BaseBoard", "ChairRail" or "Casing". The moldings can be created in the XZ (as done in the Plan view in Solid) or XY (as done in the Wall elevation view in Solid). There needs to be a minimum of two coordinate pairs to define a molding shape.

The Molding definitions should consist of 8 comma separated fields followed by a comma separated list of additional coordinate pairs as follows:

The Molding axis can be either "XZ" to specify Molding laid out in Plan view or "XY" to specify Molding applied to the face of a Wall.

If the axis is specified as "XZ", the 4th field specifies the Elevation of the Molding. If the axis is specified as "XY", the 4th field specifies the offset of the Molding from the Wall Face.

Catalog Section

The entries in this section reflect the Door and Option Property Pages for the Catalogs and Cabinets in Solid. They are used to define the Catalog, Door Styles and options for the following [Cabinet] section. This section should always contain the Catalog Name, Base and Wall Door Styles and a list of the options to apply to the Cabinets in the following [Cabinets] section. The Cabinets listed in the following [Cabinets] section will be extracted from the specified Catalog by their nomenclature and have the listed Door Styles and Options applied to them. If the Catalog, Door Style or any of the options are different for a Cabinet then a new [Catalog] section should be created for the Cabinet and the Cabinet should be placed in a following [Cabinets] section.

Parameters Section

The Parameters section reflects cabinet “Note” and “Attribute” parameters. The parameters will only be applied to the cabinet specified by the following cabinet record. Once a cabinet has been processed the Parameters list will be cleared. The Parameters section may be inserted in a list of cabinets. For example, to insert parameters on cabinet 2 in the following list…

[Cabinets]

1,"2 Door",30.00000,34.50000,24.00000,"P","R",1,""

2,"2 Door",30.00000,34.50000,24.00000,"P","R",1,""

3,"2 Door",30.00000,34.50000,24.00000,"P","R",1,""

The list becomes…

[Cabinets]

1,"2 Door",30.00000,34.50000,24.00000,"P","R",1,""

 

[Parameters]

Attribute="My Dimension","MYDIM","meas",26.75

 

[Cabinets]

2,"2 Door",30.00000,34.50000,24.00000,"P","R",1,""

3,"2 Door",30.00000,34.50000,24.00000,"P","R",1,""

Parameters are defined in the format “Type=Description,Name,ValueType,Value”

Type Attribute or Note (String)
Description Parameter Description (String)
Parameter Parameter Name (String)

You can also define Choice Lists. See the Parameters topic for more information.

Extended Format Sample

The following sample is an example of the extended ORD format that provides for the creation of Walls, Floors and adding Cabinets to the Walls. The Cabinets are visible in layout as they are added to the Walls and not to the order entry list which is the case in the standard ORD format.

The Header section reflects the current Job Door Style, Materials and Construction as well as the Wall and Floor Textures. There is also a setting to set the initial Camera position for the 3D perspective view. The [Walls] section creates three connected Walls while the [Floors] section creates two floor shapes. The [Moldings] section creates some Crown Molding in plan view at an elevation of 92 inches. It also creates some Chair Rail along the Wall in plan view followed by some Chair Rail on the Wall face in elevation view. The [Lights] section sets up three lights, one of each type allowed in Solid (it is good practice to always include an Ambient light). The [Catalog] section describes the Catalog Name, Door Styles and Options for the two Cabinets pulled from "Generic". The two Cabinets are listed in the corresponding [Cabinets] section.
