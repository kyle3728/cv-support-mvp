Custom Order Import

When opening a .TXT or .CSV file you will be presented with the custom Data Field Configuration Utility. This utility examines the text file that you are importing and helps you ensure that the data is imported in the way that you want. You can use this Utility to define what each field in your Text file represents so it is imported properly.

Toolbar

There are four options in the toolbar:

Save

Allows you to save the current configuration for re-use.

Select Configuration

Allows you to select which of your predefined configurations to use on the current Text file.

Rename

Allows you to rename the currently selected Configuration.

Delete

Allows you to delete the currently selected Configuration.

General Options

There are five general options available to configure your Text file:

Fields - This option allows you to establish how the fields in the Text file are defined. If items in the text file are separated by tabs, colons, semicolons, spaces, or other characters, select Delimited; then select what the delimiter is (Tab, Comma, Semicolon, Space, Other). If all of the items in each column are the same length, select Fixed width.

Catalog - Select the CABINET VISION Catalog to utilize.

Text Qualifier - Select the character that encloses values in your text file. When Solid encounters the text qualifier character, all of the text that follows that character and precedes the next occurrence of that character is imported as one value, even if the text contains a delimiter character. For example, if the delimiter is a comma (,) and the text qualifier is a quotation mark ("), "Tuscaloosa, Alabama" is imported into one field as Tuscaloosa, Alabama.

Dimensions - Select whether the file you are importing is Inches or Metric.

Start Import at Line - Type or select a line number to specify the first line of the data that you want to import.

Import Preview Area

In this are you will see all of the columns that CABINET VISION has created for your Text file based on the translation of the Text file. Click the column header  of the column(s) that you do not want Ignored and set its type. There are thirteen options:

Ignore - Will ignore this column when importing.

Catalog - The Catalog to pull the Object from. The name specified in the Text file must exactly match the Catalog name in Cabinet Vision.

Comment - The Objects Comment.

Depth - The Objects Width.

Description - The Objects Description.

Height - The Objects Height.

Hinge - The Objects Hinge orientation.

* = Undefined

P = Cabinet has a pair of doors (Ignored*)

L = Cabinet is left hinged (If assembly is saved with a single door L vs R will control hinge direction)

R = Cabinet is right hinged (If assembly is saved with a single door L vs R will control hinge direction)

T = Cabinet is top hinged (Ignored*)

B = Cabinet is bottom hinged (Ignored*)

 

Ignored* - Changing this value will not change the hinge type on the saved catalog object.

ID - The Objects line item number.

Finish - The Objects End Type.

* = Undefined

B = Cabinet has both ends finished

L = Cabinet has left finished end

R = Cabinet has right finished end

N = Cabinet has both ends unfinished

Name - The Objects Name.

Price - The Objects Price.

Quantity - The Objects Quantity.

Width - The Objects Width.
