S2M Part List

To use the S2M Part List CAD Import Link you will need to get the Import Link License added to your License by from your local S2M CENTER supplier.

If you are using a 3rd Party CAD Application, other than those predefined for the S2M CENTER, to create the DXF files you want to process then the S2M Part List is the Scheme you need to have selected prior to importing DXF files.

When you output your DXF's from your CAD Application you can make the Batch Import of these DXF's more seamless by creating a CncRun.txt file (See the Creating an S2M Part List topic for more information). Simply change the File Type in the Open Dialog to S2M Part List (CncRun.txt).

S2M Part List CncRun.txt File Structure

Following is the format for the S2M Part List text file.

Typical Header Layout

^Job : Test Job 1

^Description: My First Test Job

^Date: 08-18-2010 (01:03 PM)

^Parts: 5

^Unit: IN

Banding,E,Oak EB,Honey,.01

 

The Banding listing is only necessary for defining Banding coming from Cabinet Vision if you are mixing an Imported List of files with existing Cabinet Vision Parts. This section is no longer part of the S2M Part List Creator application and would need to be added manually. The S2M Part List Creator only defines S2M CENTER Material Catalog Banding through Banding Indicators.

Header Layout Field Contents

Job: Text

Description :Text

Date: Date

Parts: Indicates Number of Part Line Items in the File

Unit: Can be either IN (Inches) or MM (Metric)

1 Banding (the word "Banding" must be the first field when defining Banding)

2 Banding Code (A single letter code you use to identify the Edgeband used to build the Banding Flag described below)

3 Banding Material

4 Banding Color

5 Banding Thickness

 

Typical Part Entry

LeftEnd.dxf, End Left, 12, 30, 0.75, 3/4 Melamine, 1, 1, 0, b, 0,EEEE, 1,.5,.5

Part Entry Field Contents

1 DXF filename

2 Part description

3 Part width (x)

4 Part length (y)

5 Part thickness (z)

6 Part material name

7 Cabinet or unit number

8 Quantity to be generated

9 Grain flag (0=no, can rotate, 1=yes, cannot rotate)

The Grain flag is by Material which means a Grain and Non-Grained Material must exist and be assigned for Parts where this setting is different.

10 Face Identifier (b=Back Face f=Front Face)

11 Face Dependency (0=non face dependent, 1=face dependent)

12 Banding Flag (Optional field containing the banding indicators for each edge.)

See the Banding Flag section of the Designing Label-IT Labels topic for more information.

13 Grain Match Group (Leave at 0 (or blank) if not grain matched.  This number can begin at an index of 1.)

When imported this number may be modified so it doesn’t conflict with any parts that already have a grain match group.

14 Grain Match Position X (Establishes the X Position of the Part within the Grain Match Group)

This field is required to have a valid value if Grain Match Group has a number greater than 0.

15 Grain Match Position Y  (Establishes the Y Position of the Part within the Grain Match Group)

This field is required to have a valid value if Grain Match Group has a number greater than 0.

 

Importing Parts with no DXF File

The CAD Import routine does not require a DXF file too import a part. If no DXF file is specified it will assume the part has no operations and will import it as a rectangle.
