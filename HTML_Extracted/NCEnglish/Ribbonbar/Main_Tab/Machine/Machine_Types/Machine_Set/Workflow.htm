Workflow

Objectives

  1. Allow for different work flow scenarios where operations may be milled on multiple machines.

  2. Report work flow for a particular part.

  3. Disable warnings of an operation could not be performed on a part if it can be picked up by another machine down the chain.

  4. Provide a fail-safe program for damaged parts.

Manual (non-Machine Set) Workflow

In a Manual (non-Machine Set) Workflow scenario, operations start at the Primary Machine (Saw/Nest). If an operation can be performed on the machine then it is marked as being complete and is not available on the next machine. This works great with the following machine setup.

The following shows how the operations would be divided for this part with the nest creating the outline and vertical operations. The D&D taking the boring on the cardinal edges and the P2P getting whatever is left over.

Work Flow Option #1a

More than one P2P machine available: In this scenario, after the part is cut at the Saw or Nest the part could be brought to either P2P for finishing. All operations that were still present after the Saw/Nest would be performed on either machine:

If the operations are completely consumed prior to the next machine the work flow would be complete

Work Flow Option #1b

This is a similar scenario except that the D&D machine is used prior to the P2P

In this scenario if there are no D&D operations, that machine is skipped.

Work Flow Option #1c

Another variation of option #1 with multiple D&D machines, you could use either one

Work Flow Option #1d

Final variation of option #1: Multiple D&D and P2Ps available. You can decide where to mill the part based on current machine availability.

Work Flow Option 2

This scenario account for having a line of D&D machines, each doing specific tasks. Unlike the previous scenario where the P2P operations went to each machine, in this case the D&D operations would be consumed by a specific machine and not repeated on another.

Work Flow Option 3

In this scenario maybe you never want to mill any bore holes on the Nest machine. You want to do all boring on the secondary. However, all other operations are to be performed by the primary machine.

When the S2M Center encounters a situation like this where operations have been designated on a machine that is further down the work flow chain, we don’t want any error/warning messages about the operations not being performed on the Nest machine.

User Interface

The Machine Set interface will show the order in which the Machines would be processed. Each Machine would decide if the operations are consumed by that Machine or pass-through to the next Machine. These two icons is used to give you some feedback as to whether the operations being consumed or pass-through:

The icon on the left indicates a pass-through. Any operation performed on this Machine will still be available on the next Machine.

The icon on the right indicates a stop. Any operation performed on this Machine will consume the operation and it will not be available for any other Machine is the set.

All Stops: Each machine consumes any operation it performs

Since we want the operations to be valid for either P2P, we allow the operations to pass-through P2P-1, however, we consume them on P2P -2 so they don’t get passed down to the D&D.

This is close to the traditional output except it can go to either P2P machine; therefore we allow all operations to pass-through P2P-1.

If we had set the D&D as a pass-through, the horizontal boring would have been passed to the P2Ps.

By setting D&D 1 as a pass-through, you can do the horizontal boring at either D&D 1 or 2.  Note that D2D 2 is set to stop, otherwise the horizontal boring would have been passed along to the P2P machine.

This is setup so that either D&D can be used, and either P2P can be used on what is left over. Note that we put a stop on the last of each machine type.

Since each D&D does specific operations, we don’t want any pass-through to occur on the D&Ds, each has been set to a stop. However, whatever operations are left over can go to either P2P so we set the P2P-1 as a pass-through so any leftover operations from D&D would be available for P2P-2 as well.

Even though we intend to have the boring output on the P2P we DO NOT want to set the Nest as a pass-through, this would cause all the other operations (route, dado, and outline) to pass-through to the P2P as well. So this would be a typical “no pass-through” setup with the boring holes assigned to the P2P machine by assigning them to a ToolID that was only available on the P2P.

Special Situations

In a modification to the above scenario, we have the boring assigned to a ToolID that is available on each P2P machine. We have set both P2P1 and P2P2 as pass-through so all operations are available on any of the P2P machines.

In this scenario we have setup some special large boring tools on P2P 1. We want those operations done with those boring bits and we don’t want them passed along to P2P2 & P2P3 so we set a stop after P2P1.

The Safety Net

This scenario has a normal workflow, but it has added a safety net by 1st outputting to a “Full Outline” P2P machine that is set to pass-through. Since it is a pass-through all operations will go onto the nest as normal, but if the part is damaged for any reason, a new part can be cut to an approximate size and using the barcode for the 1st P2P machine you can completely Re-Mill the damaged part.

Pitfall – Not all paths produce the same results

The S2M Center can anticipate scenarios where due to the machine limitations an operation can only be performed on one of the two P2P machines. In this case you really wouldn’t have the option to take the part to either P2P machine.
