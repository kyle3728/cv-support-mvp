Accu-Systems MTH

Output for the Accu-Systems MTH.

Output for Accu-Systems MTH frame Mortise/Tenon/Haunching machine thru AccuCom.  This machine can haunch stiles and rails.  Output is one program per job. Tools should be assigned to this machine then linked to an operation.

Specifications for Accu-Systems MTH variable

The MTH machine manufactures face frame and door components.  These components are relational in their intersections, and units.  This relational information is usually not provided in the standard CSV exports provided by design software.  This document is designed to outline the data, and format that is needed from the design software in order to process the data for proper machining on the MTH machine.

A normal record from a CSV file would appear as follows:

Description,Width,Thickness,Length,Qunatity,Job,Material,Assembly,Comment1,2,3

Stile,2.5,0.75,29.5,4,JOB,Red Oak,Unit1,1,0,Profile1

The record provides items such as width, length, and other component specific data used for cutting and labeling.  What is missing is any data related to an intersecting part like you would find in a door, or a face frame.  Also profiling data for the sides of each part is required to detail the component.  For design software to coordinate this data, additional information is needed.  A new variable that breaks down all relating intersecting parts, and profiling can be added to provide this data.  It requires several things:

  1. Unique Unit names for assemblies per job
  2. Unique part names per unit
  3. Center point intersection
  4. Side distinct referencing
  5. Profile name for Side 0
  6. Profile name for Side 1
  7. Profile name for End 0
  8. Profile name for End 1

An example of this would be:

Stile-1,2.5,0.75,29.5,4,JOB,Red Oak,Unit1,1,0, Profile1,(1~2~Rail-1|1~28.375~Rail-2),ProfS0,ProfS1,ProfE0,ProfE1

Stile-2,2.5,0.75,29.5,4, JOB,Red Oak, Unit1,1,0, Profile15,(0~2~Rail-1|0~28.375~Rail-2),ProfS0,ProfS1,ProfE0,ProfE1

Rail-1,4,0.75,52.875,2, JOB,Red Oak, Unit1,1,1, Profile1,,ProfS0,ProfS1,ProfE0,ProfE1

Rail-2,2.5,0.75,52.875,2, JOB,Red Oak, Unit1,1,1, Profile1,,ProfS0,ProfS1,ProfE0,ProfE1

In this example, each part within the unit is defined as Stile-1 or Stile-2 etc. within Unit1.  The last field (0~2~Rail-1|0~28.375~Rail-2) references:

  1. The side of the part where the intersection occurs
    1. 0 references the left side of a vertical part
    2. 0 references the top of a horizontal part
    3. 1 references the right side of a vertical part
    4. 1 references the bottom side of a horizontal part
  2. The center point of the intersection
  3. The part name of the intersecting part
  4. “|” separates the intersections within the field

In this way complex face frames can be manufactured quickly and easily.  The following section demonstrates in detail.

With this example there are two stiles, rails, mid-rails, and one mullion.  The naming nomenclature to make the two parts unique is unimportant as long as the names are different.  In this example they will be specified with a “-1” or “-2”.

The left stile would be detailed in the CSV as follows:

Stile-1,2.5,0.75,29.5,1,JOB,Red Oak,Unit1,1,0, Profile1,

(1~1.25~Rail-1|1~28.25~Rail-2|1~13~MidRail-1|1~18~MidRail-2),ProfS0,ProfS1,ProfE0,ProfE1

Since all of the intersections of Stile-1 are on the right side, all references are “1”.  The rails (Rail-1-Rail-2) in this instance are 2.5 inches wide causing the intersection to be centered at 1.25 inches and 28.25 respectively.

The right stile would be detailed in the CSV as follows:

Stile-2,2.5,0.75,29.5,1,JOB,Red Oak,Unit1,1,0, Profile1,

(0~1.25~Rail-1|0~28.375~Rail-2),ProfS0,ProfS1,ProfE0,ProfE1

The Mid-Rail has no intersections so its record would simple be void of the variable:

MidRail-1,2.5,0.75,17.875,1,JOB,Red Oak,Unit1,1,0, Profile1, ,ProfS0,ProfS1,ProfE0,ProfE1

The Mullion would be defined much like the Right Stile:

Mullion-1,2.5,0.75,24.5,1,JOB,Red Oak,Unit1,1,0, Profile1,

(0~10.5~ MidRail -1|0~15.5~ MidRail -2) ,ProfS0,ProfS1,ProfE0,ProfE1

The Top and Bottom Rails in this case would be identical with the exception of machining side orientation:

Rail-1,2.5,0.75,52.5,1, JOB,Red Oak,Unit1,1,0, Profile1,( 1~20.375~Mullion-1),ProfS0,ProfS1,ProfE0,ProfE1

Rail-2,2.5,0.75,52.5,1, JOB,Red Oak,Unit1,1,0, Profile1,( 0~20.375~Mullion-1),ProfS0,ProfS1,ProfE0,ProfE1
