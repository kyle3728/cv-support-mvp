Schelling HPO

Creates a customized part list for Schelling's High Performance Optimizer.  Custom text fields can be setup in the machine parameters.  For the complete list of variables, as well as how to format them, please look for 'Schelling HPO' in the S2M Center help files.

Parameters

Remove Line/Position

This option allows you to remove the Positionnumber and Linenumber fields that are put at the end of the F 01 section.

Output Board Database

This parameter allows you to output additional files for output:

  • .pdb

  • .pla

These files are added in addition to the .stk file

Custom Text F[X]

This allows you to provide additional information to the .stk file in the given section.  You can use either literal text, or you can pull from a limited list of variables to display information.

A variable is formatted as follows:

{variable name}

As an example, if you wanted to pull the Front Barcode Program to a section, you would do the following:

Custom Text F10: {FrProgram}

In the .stk file, this would give a result similar to:

F 10 r008001

The list of available variables is:

  • PartID: Reference to [Parts.ID] linking the record to a part record

  • GroupID: Reference to [Groups.ID] linking the part to a group

  • FrProgram: Program name for the part front face operations

  • BaProgram: Program name for the part back face operations

  • FrMirror: Face operation are in a mirrored work field

  • BaMirror: Back operation are in a mirrored work field

  • FrBarCode: Front program bar code

  • BaBarCode: Back program bar code

  • FrWorkZone: Front program work zone

  • BaWorkZone: Back program work zone

  • LeBarCode: Bar code for horizontal machining on left edge

  • RiBarCode: Bar code for horizontal machining on right edge

  • ToBarCode: Bar code for horizontal machining on top edge

  • BoBarCode: Bar code for horizontal machining on bottom edge

  • IntBandBarCode:Bar code for interior banding

  • ExtBandBarCode: Bar code for exterior banding

  • Image: Binary graphic for this part used in reports

  • ImageName: File title of part/label image

  • RunMargin: Margin around the part for nesting

  • RunPrimaryEdge: Primary part edge that rest against the point-to-point pins (1=left, 2=top, 3=right, 4=bottom)

  • RunQty: Quantity of the part in this MachineSet Run

  • CutWidth: Part width as cut on machine.  Extended size: which includes pre-mill + EDX & EDY

  • CutLength: Part length as cut on machine.  Extended size: which includes pre-mill + EDX & EDY

  • Orientation: 0x01 ArrowLe, 0x02 ArrowRi, 0x04 ArrowUp, 0x08 ArrowDn, 0x10 ArrowFlip, 0x20 Face, 0x40 2nd Program, 0x80 Spare (1st 8 Bits 0xff for Left/Face)(Next 8 bits 0xff00 for Right/Back)(Next 8 bits 0xff0000 for Top)(Next 8 bits 0xff000000 for Bottom)

  • BandedWidth: Finshed part width after banding. Size include banding

  • BandedLength: Finshed part length after banding. Size include banding and EDX & EDY

  • Parts.ID: Unique Primary Reference

  • Parts.JobID: Reference to [Jobs.ID] linking part to a job

  • MatDBID: Material identifier

  • Parts.Name: Name for the part

  • InternalName: Internal part name

  • Optimize: Part has been set for optimization

  • CNC: Part has been set for CNC output

  • PrimaryEdge: Primary part edge that rest against the point-to-point pins (1=left, 2=top, 3=right, 4=bottom)

  • Outline: Shape in an ASCII comma delimited string. Format:(node count,x1,x2 ,radius1,flags1,.....xN,yN,radiusN,flagsN)  Flags:(0x08=VTX_Radius,

  • OutlineToolID: Reference to [Tools.ID] linking a tool to use for the

  • GangBore: Part has gang boring

  • HorizontalBore: Part has horizontal boring

  • PriMachineID: Reference to [Machines.ID] linking this part to the machine used for primary output

As a note, if more information is added to the PartRunData table, more information will be populated into this list as well.

Variables are case insensitive, meaning that {FrProgram} works just like {frprogram} or {FrPrOgRaM}.  You can also use multiple variables in one section and mix in literal strings as well:

{FrProgram} / {FrBarCode}

The user has 19 sections that they may enter fixed text/variables into the output (F10 – F29)

Notes for Grain Matching

When using the ‘Close grain match gaps larger than x’ preference in the S2M Center (under the Geometry tab), you may get additional cut passes for doors that sit under drawers, or in other situations.  This output will use the Kerf width (Cross and Rip) to determine what the gaps between doors and drawer should be and you can adjust those values to get the output you need to remove those additional passes.
