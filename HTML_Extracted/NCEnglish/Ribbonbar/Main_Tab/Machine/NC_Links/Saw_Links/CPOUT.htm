CPOUT

This is a Panel Saw Link.

This link creates a standard CPOUT file with additional formatting conventions as outlined by CADCode. Following is the file specification:

Pattern Export File Specification (CPOUT)

The first three lines describe the header information as follows:

The HDR1 line contains the cutlist number, Job name and Materials in the following format:

HDR1,cnum,jobmat

HDR1             is the line descriptor

cnum             indicates the cutlist number, a string with leading zeros included.

jobmat           indicates the cutlist title, an ASCII string of length 50 or less, composed of the Material Name and Job Information.

The HDR2 line contains information regarding the date and time the cutlist was optimized in the following format:

HDR2,date,time

HDR2             is the line descriptor

date                indicates the date of optimization. It is an ASCII string of length 12. [Dec 08, 2006]

time                indicates the time of optimization. It is an ASCII string of length 8. [01:12 PM]

The HDR3 line contains information about the product that produced the pattern data file in the following format:

HDR3,bnum,ver,pcode,sawid

HDR3             is the line descriptor

bnum             indicates the Build number of the product used. It is an ASCII string of length 6.

ver                  CPOUT Version [2.01]

pcode            indicates product code [CVI]

sawid             indicates the SawLink ID used [0009]

The next three lines describe the controls used in the pattern making process.

The CTL1 line contains the dimensional unit of the cutlist, the output unit, the grain indicator, area reporting, and saw type in the following format:

CTL1,unit1,unit2,grain,areafac,sawtype

CTL1              is the line descriptor.

unit1               indicates dimensional Unit of Measure. It is a one character field as follows:

                                    F = Fractional Units

                                    D = Decimal Inches

                                    M = Millimeters

unit2               indicates cutting instruction Unit of Measure. It is a one character field as described for unit1 and also always matches unit1.

grain              indicates the graining control. It is a one character field as follows:

                                    Y = Cutlist is grained

                                    N = Cutlist is not grained

areafac          indicates the area unit throughout the file. *Planit always outputs 1. 1 = SqFT in Imperial and SqM in Mtric based on the unit1 setting.

sawtype        indicates the type of saw specified. It is a one character field as follows:

                                    0 = no saw type specified

                                    1 = single axis saw

                                    2 = two axis saw with single feeder

                                    3 = two axis saw with multiple feeder

The CTL2 line describes dimensional content in the following format:

CTL2,rkerf,xkerf,maxsw,minsw,pwadj,pladj

            CTL2              is the line descriptor

            rkerf               indicates the rip cut kerf

            xkerf               indicates the cross cut kerf

            maxsw           indicates the maximum strip section width

            minsw            indicates the minimum strip section width

pwadj            indicates the part width adjustment. *Planit always outputs 0.

pladj               indicates the part length adjustment. *Planit always outputs 0.

The CTL3 line describes the Front Trim value in the following format:

CTL3,ftrim

            CTL3              is the line descriptor

            ftrim                indicates the value for the Front Trim.

The THK1 line specifies the sheet stock thickness and stack (book) height in the following format:

THK1,sthick,stackh

            THK1             is the line descriptor

            sthick indicates the sheet stock thickness

stackh           indicates the maximum stack height. This is a measurement not the number of sheets.

The next two lines describe cutting statistics.

The STA1 line specifies the total number of cuts, grouped by each phase type, as indicated below. Each field is right justified and has a length of 6 digits, and is in the following format:

STA1,nrcut,nxcut,n3cut

            STA1              is the line descriptor

nrcut              indicates the number of rip cuts in the cutlist and includes the leading trim cut in the calculation.

nxcut             indicates the number of cross cuts in the cutlist and includes the trim cut. If there are identical strips then the cross cuts are not counted on the additional strips.

n3cut             indicates the number of third or greater phase cuts in the cut list.

The STA2 line specifies the total distances cut, grouped by each phase type. The distances are a function of the dimensional units (unit1). If the dimensional units are fractional or decimal inches, the distances are in feet, If the dimensional units are millimeters, the distances are in meters. Each field is right justified to eleven characters and has the form ‘xxxxxxx.yyy’.

The STA2 line has the following format:

STA2,drcut,dxcut,d3cut

            STA2              is the line descriptor

drcut              indicates the total distance of rip cuts in the cutlist including the leading trim cut.

dxcut             indicates the total distance of cross cuts in the cutlist including the trim cut. Unlike the nxcut field in STA1, the cross cut distance does not skip identical strips.

d3cut             indicates the total distance of third or greater phase cuts in the cutlist.

The next three lines describe sheet stock records.

The INV1 record for each sheet stock item has the following format:

INV1,supply,used,ucost,width,length,trim1,trim2,trim3,trim4

            INV1               is the line descriptor

supply           indicates the available supply of the sheet. It is a right justified numeric field of length 5. A supply of ‘*****’ indicates an infinite supply.

used               indicates the quantity of the sheet used in the cutlist. It is aright justified numeric field of length 5.

ucost             indicates the unit cost per square foot/meter. It is a right justified field of length 9 in the form ‘xxx.yyyy’.

width              indicates the width of the sheet.

length            indicates the length of the sheet.

trim1               indicates the leading width trim.

trim2               indicates the leading length trim.

trim3               indicates the trailing width trim.

trim4               indicates the trailing length trim.

The INV2 record for each sheet stock item has the following format:

INV2,desc,areacut,finparts,lpfin

            INV2               is the line descriptor

desc               indicates the description of the sheet. It is a character field of length 24.

            areacut          indicates the total area cut of the sheet. [xxxxxxxxx.yyyy]

finparts          indicates the total area finished (parts in sheet). [xxxxxxxxx.yyyy]

lpfin                indicates the total low priority area finished. *Planit always outputs 0.

The INV3 record is used to describe extra information about the sheet stock and has the following format:

INV3,vendor,location,ori

            INV3               is the line descriptor

vendor           indicates the vendor from which the sheet stock was purchased. *Planit does not use this field.

location         indicates the physical location in the shop for the sheet stock. *Planit does not use this field.

ori                   is the orientation that the user prefers for the first cut on the sheet. It is a one character field as follows:

                                    0 = don’t care

                                    1 = first cut along length

                                    2 = first cut along width

The next two lines describe part cut sizes.

The ORD1 record for each cut size has the following format:

ORD1,demand,supply,width,length,prio,grain

            ORD1             is the line descriptor

demand         indicates the quantity requested for the cut size. It is a right justified field of length 5.

supply           indicates the quantity supplied/produced of the cut size. It is a right justified field of length 5.

            width              indicates the width.

            length            indicates the length.

prio                 indicates the priority of the part. It is a one character field where:

H or Blank indicates high priority (NcCenter always outputs an H

                                    L indicates low priority

grain              *Planit leaves blank because it is indicated in CTL1.

The ORD2 record for each cut size has the following format:

ORD2,pcost,desc1,desc2

            ORD2             is the line descriptor

            pcost             cost per part to 1/10 of a cent. [dddd.ccc]

desc1             indicates the primary part description. *Planit leaves blank because it is described in PRT

desc2             indicates the secondary part description. *Planit leaves blank because it is described in PRT

The ORD3 record is used to access the part information for each cut size and has the following format:

ORD3,nparts,index

            ORD3             is the line descriptor

nparts            This is the number of parts that have been combined into this cut size.

index              This is the index into the list of parts where this cut size begins.

The next five records are for the parts that make up the above cut sizes.

The PRT1 record for each part has the following format:

PRT1,dmnd,ordno,width,length,dolabel,prtname,jobname,partcom,

cncprg

            PRT1             is the line descriptor

            dmnd             is the quantity the user specified for this part

            ordno             is the index or offset into the cut sizes specified above

            width              is the width for this part

            length            is the length for this part

dolabel          indicates if user wants a label made for this Part. *Planit always outputs 1 (True).

            prtname        is the name of this part

            jobname       is the name of the Job this part is in

            partcom        is the name of the Label Graphic for Real-Time Label printing

            cncprg           CNC Program Name – 07/06/06(date) – Primary Barcode

The PRT2 record for each part has the following format:

PRT2,postlam,secprog,roomname

            PRT2             is the line descriptor

postlam         indicates post laminating instructions. *Planit always outputs 0.

            secprog      is the Secondary CNC Program Name

            rmname         is the name of the Room the part is in

The PRT3 record for each part has the following format:

PRT3,edges,priprog,wmatl1,wcolor1,prizone,lmatl1,lcolor1,secprog,wmatl2,wcolor2,seczone,lmatl2,lcolor2

            PRT3             is the line descriptor

            edges            is the edge banding codes for the Part [W1,L1,W2,L2]

0 = No Banding

1 = Door Banding

2 = Exterior Banding

3 = Interior Banding

4 = BandingB

5 = BandingC

6 = BandingF

7 = BandingG

8 = BandingH

9 = BandingJ

Other Banding letters are ignored as CPOUT only has placeholders for 0 - 9 identifiers.

            priprog          is the Primary CNC Program Name

            wmatl1           is the Edge Band Material for W1

            wcolor1         is the Edge Band Finish Color for W1

            prizone          is the Primary CNC Program Zone

            lmatl1 is the Edge Band Material Name for L1

            lcolor1           is the Edge Band Finish Color for L1

            secprog        is the Secondary CNC Program Name

            wmatl2           is the Edge Band Material for W2

            wcolor2         is the Edge Band Finish Color for W2

            seczone        is the Secondary CNC Program Zone

            lmatl2 is the Edge Band Material for L2

            lcolor2           is the Edge Band Finish Color for L2

The PRT4 record for each part has the following format:

PRT4,dname,cwidth,cheight,cdepth,ddesc,comments

PRT4             is the line descriptor

            cname           is the name of the Cabinet this part is in

            cwidth           is the width of the Cabinet this part is in

            cheight          is the height of the Cabinet this part is in

            cdepth           is the depth of the Cabinet this part is in

            cdesc             is the description of the Cabinet this part is in

            comments    are the comments on the Cabinet this part is in

The PRT5 record for each part has the following format:

PRT5,prio,grain,qty,qtyover

            PRT5             is the line descriptor

            prio                 is the priority which is always set to high (H)

grain              indicates grain [Y or N] – Blank indicates use the global setting as defined in CTL1

            qty                  this parts quantity

            qtyover          this parts quantity plus overmake

Following the part records are the drop records. There is one record for each drop. The number of the drop items what will appear in the file is specified in the second line of the file as described above.

The format of the drop record is as follows:

OFF1,supply,width,length

            OFF1              is the line descriptor

            supply           is the quantity of the drop

            width              is the width of the drop

            length            is the length of the drop

Following the drop records are the pattern description records.

For each pattern in the file there are five header records. Following the header records are the cut type records.

The five pattern records are as follows:

The first pattern record has the following format:

PAT1,invn,pat_type,repeat,cycles

            PAT1              is the line descriptor

invn                indicates the sequence number of the sheet stock item used to cut the pattern, It is a string length of 2. with the leading zeros included.

pat_type        indicates a one character field indicating the type of pattern as follows:

                                    L = Long rip pattern

                                    S = Short rip pattern

                                    H = Head cut pattern

            repeat            indicates the number of sheets in the book (stack)

cycles            indicates the number of cycles (books/stacks cut). i.e. If you have 11 sheets, but the book height is 3 sheets, the cycle would be 4 indicating (3 + 3 + 3 + 2). With the NcCenter’s current output the cycle is always 1 because the NcCenter pre-groups the materials by maximum book (stack) height.

                                NOTE: A long rip pattern indicates that the rip cuts are made parallel to the larger dimension of the sheet. i.e. If the width dimension entered is less than the length dimension entered for the sheet, this indicates that the rip cuts are parallel to the length dimension of the sheet, But, if the width dimension is larger that the length dimension, the rip cuts are made parallel to the width dimension entered.

                                A short rip pattern indicates that the rip cuts are made parallel to the shorter dimension of the sheet. i.e. If the width dimension entered is less than the length dimension for the sheet, this indicates that the rip cuts are made parallel to the width of the sheet. But, if the width dimension is larger that the length dimension, the rip cuts are made parallel to the length dimension.

                                A head cut pattern has the first cut across the width of the sheet (like a short rip) but after the initial head cut the rest of the pattern looks like a long rip. The strips within the head section are identical. The size of the head section is controlled by the cutting rules.

The second pattern header record has the following format:

PAT2,yield,uarea,uareah,uareal,cpua

            PAT2              is the line descriptor

            yield               indicates the percentage of yield

            uarea             indicates the used area per panel

uareah           indicates the used area by high priority parts. *All Planit Parts are high priority so this field always matches the value in uarea.

uareal            indicates the used area by low priority parts. *All Planit parts are high priority so this field is always 0.

            cpua               indicates the cost of the used area per panel

The third pattern header record has the format below. Each field is right justified to a length of 5.

PAT3,nrcut,nxcut,n3cut,mnxcut

            PAT3              is the line descriptor

            nrcut              indicates the number of rip cuts per cycle

            nxcut             indicates the number of cross cuts per cycle

n3cut             indicates the number of 3rd and higher phase cuts per cycle

mnxcut          indicates the number of cross cuts in the worst strip in the pattern

The fourth pattern header record has the format below. Each field in the record is right justified and is 11 characters in length with the format ‘XXXXXXX.YYY’.

PAT4,drcut,dxcut,d3cut

            PAT4              is the line descriptor

            drcut              indicates the total rip cut distance per cycle

            dxcut             indicates the total cross cut distance per cycle

            d3cut             indicates the total of 3rd and higher phase cuts per cycle

The fifth pattern header record has the following format:

PAT5,ton,toff,con,coff

            PAT5              is the line descriptor

            ton                  indicates the on-line time in minutes for all cycles

            toff                  indicates the off-line time in minutes for all panels

            con                 indicates the on-line operating cost

            coff                 indicates the off-line operating cost

Following the PAT5 record will be one of the following record types.

Patterns that have head cuts are divided into two sections. We call these sections the “head section” and the “body section”. The cutting instructions (the CUT1 records) do not change for the head section, they still have codes for rip cuts, cross cuts, and 3rd and higher phase cuts. The head section will be preceded by a record that starts with the keyword “HCUT”, followed by the CUT1 records. At the end of the head section (or at the start of the body section) you will have a record that starts with the keyword “BCUT”, followed by the CUT1 records for this part of the pattern.

HCUT

The presence of this record means that this pattern has a head section and the CUT1 records for the head section will follow this record.

Note that this record is OPTIONAL and will NOT appear for patterns that do not have head cuts.

BCUT

This record will precede the CUT1 records for the pattern. If the pattern contains head cuts then this record will precede the CUT1 records for the body section.

Following the above HCUT or BCUT records are pattern description records, which indicate how to cut the sheet. This record has the following format:

CUT1,level,dist,otype,onum

            CUT1             this is the line descriptor

level               indicates the phase number of the cut. It is a two character field with leading zeros included, i.e.,

                                                01 = rip cut

                                                02 = cross cut

                                                03 = 3rd phase cut

                                                04 = 4th phase cut, etc.

            dist                 indicates the dimensional distance of the cut

            otype             indicates the part type. It is a one character field where:

                                                0 = Part not produced with cut

                                                1 = Scheduled part

                                                2 = Drop part

onum             indicates the sequence number of the size produced with this cut. If additional cuts are required before a cut size is produced with a cut, a zero is supplied for onum. Thus a value of ‘5’ means that this cut results in a finished cut size whose index is ‘5’ in the ORD records (the first ORD record is ‘1’).

NOTE: Instructions are not provided for leading trim cuts. They must be determined from the corresponding sheet stock item and the head cut trim setting.

The following indicates the end of the file:

END
