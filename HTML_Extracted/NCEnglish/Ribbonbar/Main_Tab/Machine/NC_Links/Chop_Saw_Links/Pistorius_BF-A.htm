Pistorius BF-A

This is a Chop Saw Machine Link.

Output for Pistorius BF-A CNC Sawing and Notching (Haunching) Machine. Only parts made of board stock are output. Output can include machining mortise and tenons for stiles and rails when operations are linked to parts using a specific tool (see the eSupport Tech Packages area for examples of using this NcLink with Cabinet Vision Solid).

Valid feed rates for the tool are 1 - 5 in/sec.

 

Output File Specifications

The are 3 sections in the file, Cabinets, Parts and Materials:

[Cabinets]

The Cabinets section is not used.

[Parts]

The Parts section has the following structure:

PartID - Unique identifier for the part (Data type: long integer)

CabinetID - Not used. Must be 0.

PartName - Name of part (not the part type) (Data type: string 50 characters max)

MaterialID - Unique identifier from [Materials] section that defines the parts material (Data type: long integer)

PartTypeID - Identifies the part type (Data type: long integer, 1-Right style, 2-Bottom rail, 3-Left style, 4-Top rail, 5-Mid style, 6-Mid rail)

ParentID - Not used. Must be 0.

ChildID - Not used. Must be 0.

Side1Data - Notch locations (Data type: string 255 characters max, see notes below)

Side2Data - Notch locations (Data type: string 255 characters max, see notes below)

PartNumber - Not used. Must be 0.

Left - Not used. Must be 0.

Right - Set to length of part not including extra length required due to bead (Data type: string 16 characters max)

Bottom - Set to length of part not including extra length required due to bead (Data type: string 16 characters max)

Top - Always set to zero for your import file

Notes on format of Side1Data and Side2Data:

Each notch on the side has the following format

";MaterialID:NotchStart,NotchEnd;"

Each entry is surrounded by a semicolon. If no notching is required there will just be a ; in the field.

The MaterialID comes from Materials section with width of the material matching the width of the notch.

 

[Materials]

The Materials section has the following structure:

MaterialID - Unique identifier for the material (Data type: long integer)

MaterialName - Name of material (Data type: string 20 characters max)

NumberOfBeads - 0, 1, or 2 (Data type: integer)

Width  - Overall width of material including bead dimension (Data type: string 10 characters max)

Bead - Size of bead (Data type: single)

CutSpeed - Speed material will be cut. (Data type: string 10 characters max,1-5in/sec 3 seems to work with everything)
