Offcut Manager

The Offcut Manager is used to manage the reuse of Offcuts. Offcuts are considered any scrap/waste from an Optimized Sheet, equal to or larger than the size set in the Offcuts Tab in Optimizer Parameters or the Offcuts Tab in Material Properties.

Offcuts Tab

The Offcuts Tab display any Offcuts available for use in Jobs.

If you are using a Material Handling System  you can also select to Sync with it here.

Following are the tools available in the toolbar:

Print - Will Print the list of Offcuts.

Filter - Will allow you to Filter the Offcuts based on the field you select (Material, Width, Length, Qty) on the following conditions:

Equal to

Not Equal to

Contains

Does Not Contain

Remove Filter - Will Remove the active Filter to show all Offcuts.

Set Columns - Allows you to enable/disable and reorganize the columns.

In addition to controlling which items display you can also control the order in which the columns appear. Simply click and drag the item you want to move into its new position.

Bins Tab

In previous Versions of the S2M CENTER it was a requirement to setup Bins for Offcuts. This is now not a requirement; all Offcuts, equal to or larger than the size set in the Offcuts Tab in Optimizer Parameters or the Offcuts Tab in Material Properties will now be stored when utilizing Update Offcuts from the Patterns View.

If we look at the first bin it is setup to capture Offcuts that I can cut my 300 x 300 Upper ends from. The way the Bin is setup with any Offcuts that are sized between 300 x 300 to 609 x 456 it is going to assign for this Bin. Basically if I go to this Bin it is going to have parts that range in a size that I can get my 300 x 300 parts out of. The parts in this bin maybe larger, say 575 x 400, but the rule I am trying to establish is that it can be utilized in a future Optimization to get a 300 x 300 Upper end panel out of.

The bins are setup to contain Offcuts of at least the minimum specified size - in other words they are used to group Offcuts into similar sized panel stock. If you have a number of Offcuts in a bin and want to add them to the optimization, the smallest size of the bin dimension is used for the material size. The bin is basically saying "I have these pieces of material that are at least this size".  When you add Offcuts to the bins, it does not store the size of each part that gets added to the bin, it just increases a quantity of parts that are in a particular bin. So when adding the parts back to the Optimizer it uses the smallest dimensions of the bin. In other words a bin is a set of parts that are AT LEAST THIS SIZE even though some may have originally been bigger.

You can also Print and Set Columns on the Bins tab.
