Real-Time Labeling

When using Real-Time Labeling on your Machine (On Board Machine Labeling Solution). Following are the available Label Applicator Machines:

None

Use this option if you are not driving the Label Applicator from the S2M CENTER.

Anderson

A proprietary output produced for Anderson Machines. The Anderson output consists of the following elements:

  1. The NC Code for the nest

  2. An *.lpf file for each NC Code file. This indicates the label position, based on the center of the label for each part and a rotation flag (1=0, 2=90, 3=180, 4=270). The Anderson machine has the ability to rotate the label 90 degrees so all labels images are output rotated either 0 or 180. The rotation flag then instructs the machine if it should rotate an additional 90 to cover the 90 and 270.

  3. An *.csv file for each NC Code file. The *.csv file contains additional part information that cab be place on the label. Look to the .CSV File section of this topic for a field map.

  4. An *.zpl file for each part that contains the part graphic.

The *.anc (Nc Code) *.lpf and *.csv file should all have the same file name. For example: r0201001.arb, r0201001.lpf, r0201001.csv

Biesse

A proprietary output produced for Biesse Machines. The Biesse output is much simpler than the Anderson. Unlike the Anderson there is not additional part data that needs to be output; they will rely completely on the Label IT image.

The .arb (Biesse Nc Code Files) will be modified after they are processed by adding an addition section for the label position and zpl names as follows:

[CUSTOMSEZ01]

<LABEL>

PEZZO=1

X=100

Y=200

STRCUST=C:\LBL\

FILERIS=R01P0001.ZPL

PEZZO=2

X=200

Y=200

STRCUST=C:\LBL\

FILERIS= R01P0002.ZPL

</LABEL>

%

[]

In addition the [PARAMETRI01] section will need to be modified to include:

ETC=1 KE=1024

Excerpt from Biesse Setup Documentation

ETC parameter must be set to 1 when the panel labeling must be performed using the information written in section CUSTOMSEZ01. If the parameter is set to 0, the PC will ignore the information contained in this section, even if it contains some values. When the ETC parameter is set to 1, it affects the value KE, which changes from 0 to 1024. Section CUSTOMSEZ01 contains the information regarding the labeling operation. This section must starts with <LABEL> and ends with </LABEL>. Each label to be sticked must be described in the program and the description must contain the following information:

PIECE

X

Y

STRCUST

FILERIS

The field PIECE is a progressive number of the label, i.e.: PEZZO=1 Field X is the position, along the X axis, where the label must be sticked (its number is in the PIECE field). It refers to the reference point of the applicator (the offset set in the field max_lung of the tool is added and shows the label used), for example: X=123,75 Field Y is the position, along the Y axis, where the label must be sticked (its number is in the PIECE field). It refers to the reference point of the applicator (the offset set in the field max_diam of the tool is added and shows the label used), for example: Y=361,84 The field STRCUST must show the complete path where the watcher will find the label file to print, i.e.: STRCUST=C:\ETC\ The field FILERIS must show the label file name that the watcher will find to print, i.e.: STRCUST=LABEL01.PRN

Biesse CIX

A proprietary output produced for Biesse Machines using CIX.

Robotic Automation

A proprietary output produced for Robotic Automation Machines.

.ZPL File

A .zpl file is basically NC Code for a Zebra printer. It has commands such as move to this X,Y point, and print “Job Name” with this font. Move to this X,Y point, draw a line 60 points wide…

One element of a ZPL file is an image. An image imbedded within a ZPL file indicates a block of points then indicates which points in that block will be black. We have observed 4 different methods for describing the image. The image format can be set/modified in the registry with the following settings:

HKEY_CURRENT_USER\Software\Cabinet Vision\NcCenter_12\CncOutput

ZPL_Format - Valid values are “Anderson”, “Andi”, “Biesse”, 0,1,2,3

If the integrator needs to modify the ZPL_Format he should set the ZPL_Format to -1 then use the following settings:

ZPL_AllowCount - Do not use the raw format, use the count method to compress file size

ZPL_AllowComma - Use a comma to separate each line of the ZPL file.

ZPL_AllowColon - Used with allow comma, use a colon to indicate a repeat of the previous line.

ZPL_AllowCRLF - Use a carriage return, line feed after each line of the ZPL file.

ZPL_AllowRotate - Rotate the image 0, 90, 180 or 270 degrees to match the nested part.

 

Template.ZPL

Since the ZPL file is not just the image it is the full set of instructions to the zebra printer the you can setup a template to embed your graphic data into your existing file. If a template.zpl file is found in the output directory, it will be used in the following manner:

^FX Output from Planit ^FS

[IMAGE.DATA]

^XA

^DF[IMAGE.NAME]^FS

^PRC

…

^XZ

^FX End of job

^XA

^XF[IMAGE.NAME]

^XZ

The highlighted areas will be replaced with the image data from the S2M CENTER while the rest of the ZPL file is untouched.

For rotated ZPL output the program will look for the template0.zpl, template90.zpl, template180.zpl, or template270.zpl. This will allow for different label layouts for rotated output. If the rotated template cannot be found it will look for the template.zpl.
