The PNX Import/Export Files

The PNX and SNX files are SQL backups which you can restore into an existing or new database to use. The PNX and SNX file store all Machining Information required for Importing/Exporting a Job to/from the S2M CENTER.

The following is the breakdown of the tables and their fields:

Assemblies

This table stores the Assemblies.

Field

Description

ID

A unique number to reference this assembly.

JobID

Reference to [Jobs].[ID] linking the assembly to a Job.

Number

The number for the assembly.

Name

The name of the assembly.

Width

A value representing the width of the assembly.

Height

A value representing the height of the assembly.

Depth

A value representing the depth of the assembly.

RoomID

Reference to [Rooms.ID] linking the assembly to a room.

WallNo

The Wall # the assembly is associated to.

Class

 

 

 

 

 

 

The Class of the Cabinet/Assembly:

1 = Base

2 = Upper

3 = Tall

4 = Vanity

5 = Sink

6 = Split Level
7 = Top

Type

 

 

 

 

 

 

 

 

 

 

 

The Type of the Cabinet/Assembly:

1 = Standard

2 = Corner

3 = Corner 45

4 = Corner 90

5 = Peninsula

6 = Lazy Susan

7 = Pantry

8 = End

9 = Blind Left

10 = Blind Right

11 = Filler

Description

The description for the assembly.

Comment

A user comment for the assembly.

Parameters

Parameters for the assembly.

Materials

This table stores all the Materials in the S2M Center Job.

Field

Description

ID

A unique number to reference this Material.

MatDBID

Material identifier. Combines [SourceDB] and [SourceDBID], using bit(24 + SourceDB) as a source flag.

SourceDB

The source of the Material:

0 = Unknown

1 = CABINET VISION

2 = Kitchen Builder

3 = Cabnetware Face Frame

4 = Cabnetware 32mm

SourcDBID

The source application's DBID for the Material.

Name

The name for the Material.

Description

The description for the Material.

Qty

A value representing the Quantity available of the Material.

Width

A value representing the Width for this sheet of Material.

Length

A value representing the Length for this sheet of Material.

Thick

A value representing the Thickness for this sheet of Material.

Type

The Materials Type:

0 = Panel Stock

1 = Board Stock

2 = Laminate

3 = Banding

4 = Hinges

5 = Guides

6 = Pulls

7 = Miscellaneous

8 = Tops

9 = Molding

Optimize

A Boolean value set to true if the Material can be used in a Panel optimization.

Offcut

A Boolean value set to true if this Material originated from an Offcut Bin.

SubSheet

Material represents a sub-sheet when grain matching block exceeds saw phase limitations.

Grain

A Boolean value set to true if the Material has Grain.

Cost

A value representing the Price associated with this sheet of Material.

FaceDependent

A Boolean value set to true if this Material is face dependent (single sided).

TextureFace

The Front material texture name.

TextureBack

The Back material texture name.

TextureEdge

The Edge material texture name.

MachineOverrideMatDBID

Cross-reference to [MachineOverrides].[MatDBID] linking a Material to overrides of the default parameters.

ClimbCut

A Boolean value set to true if it is better to cut this Material with a climb or conventional cut.

FeedRatePercent

A value representing the Percent of optimum Tool Feed Rate to use with this Material.

MinimizeFaceChip

A Boolean value set to true if the System needs to select a tool for operations that minimize chipping on the front face. Assists in Selecting Up Shear / Down Shear router bit.

MinimizeBackChip

A Boolean value set to true if the System needs to select a tool for operations that minimize chipping on the back face. Assists in Selecting Up Shear / Down Shear router bit.

FaceUpToolID

Cross-reference to [Tools].[ID] linking the outline tool to use if this Material is face up.

FaceDownToolID

Cross-reference to [Tools].[ID] linking the outline tool to use if this Material is face down.

PartOperations

This table stores the Part Operations.

Field

Description

PartID

Cross-reference to [Parts].[ID] linking the operation to a part.

ID

A unique number to reference this Part Operation.

Name

The Name for the operation.

Type

 

 

 

 

 

The Operation Type:

1 = Dado

2 = Line Bore

3 = Cutout

4 = Route

9 = Hole

Style

 

 

 

 

 

 

The Operation Style:

0x01 = Pocket

0x02 = Inside

0x04 = Outside

0x08 = MDF Glass

0x10 = MDF Outline

0x20 = Preserve Dir

ToolID

Cross-reference to [Tools].[ID] linking a tool to the operation. For automatic tool selection (recommended) the [ToolID] should be 0.

Width

A value representing the Width of the operation.

Length

A value representing the Length of the operation.

Depth

A value representing the Depth of the operation.

PosX

A value representing the X position of the operation.

PosY

A value representing the Y position of the operation.

PosZ

A value representing the Z position of the operation.

RotX

A value representing the Rotation of the operation about the X axis.

RotY

A value representing the Rotation of the operation about the Y axis.

RotZ

A value representing the Rotation of the operation about the Z axis.

Repeat

A value representing the Repetition for Line Bores.

Spacing

A value representing the Spacing for Line Bores.

Outline

 

 

 

 

 

 

Shape in an ASCII comma delimited string:

Format:

(node count,x1,x2 ,radius1,flags1,.....xN,yN, radiusN,flagsN)

Flags:

0x08 = VTX_Radius

0x0400 = VTX_ArcLine

0x010000 = VTX_Visible

Group

 

Used to override normal operation sort. A positive group will allow an operation to be milled after the outline on a nest. A negative group will allow an operation to be milled prior to the outline on a part.

Priority

An Override to the normal sort order of operations. Operations are 1st sorted by priority ascending.

DrawingID

Cross-reference to [Drawings].[ID] linking an extended AlphaCAM drawing with this operation.

Parameters

A List of System parameters associated with this operation.

Parts

This table stores the Parts for the S2M Center Job.

Field

Description

ID

A unique number to reference this Part.

JobID

Cross-reference to [Jobs].[ID] linking part to a job.

MatDBID

Material identifier.

Name

The Name for the part.

InternalName

The Internal part name.

Optimize

A Boolean value set to true if the Part has been set for Optimization.

CNC

A Boolean value set to true if the Part has been set for CNC output.

Width

A value representing the Width of the part + EDX value without banding.

Length

A value representing the Length of the part + EDY value without banding.

WidthString

A value representing the Width displayed in output increment and precision.

LengthString

A value representing the Length displayed in output increment and precision.

NominalWidth

A value representing the Width of part without banding or EDX value.

NominalLength

A value representing the Length of part without banding or EDY value.

Comment

The user comment for the part.

Qty

A value representing the Quantity of the part.

OverMake

A value representing the Quantity that may be over made for this part.

RoomID

Cross-reference to [Rooms].[ID] linking the part to a room.

AssemblyID

Cross-reference to [Assemblies].[ID] linking the part to an assembly.

PrimaryEdge

 

 

 

 

A value representing the Primary part edge that rests against the point-to-point pins:

1 = Left

2 = Top

3 = Right

4 = Bottom

Mirror

A Boolean value set to true if the part should be output to a mirrored zone.

Shaped

A Boolean value set to true if the Part is shaped.

Outline

 

 

 

 

 

 

Shape in an ASCII comma delimited string:

Format:

(node count,x1,x2 ,radius1,flags1,.....xN,yN, radiusN,flagsN)

Flags:

0x08 = VTX_Radius

0x0400 = VTX_ArcLine

0x010000 = VTX_Visible

OutlineToolID

Cross-reference to [Tools].[ID] linking a tool to use for the outline of this part.

GangBore

A Boolean value set to true if the Part has Gang (Multi-Drill) Boring.

HorizontalBore

A Boolean value set to true if the Part has horizontal boring.

PriMachineID

Cross-reference to [Machines].[ID] linking this part to the machine used for primary output.

SecMachineID

Cross-reference to [Machines].[ID] linking this part to the machine used for secondary output.

HorMachineID

Cross-reference to [Machines].[ID] linking this part to the machine used for horizontal edge output.

CutMachineID

Cross-reference to [Machines].[ID] linking this part to the machine used for cutting part from a strip.

PrefPriMachineID

Cross-reference to [Machines].[ID] linking this part to the preferred machine to use for primary output.

PrefSecMachineID

Reference to [Machines].[ID] linking this part to the preferred machine to use for secondary output.

Margin

A value representing the Margin around part for nesting.

HasBanding

A Boolean value set to true if the EdgeBanding table contains data for this part.

Banding

 

 

 

 

 

Four character banding code (Width1, Length1, Width2, Length2)

0 = None

1 = Door Banding

2 = Exterior Banding

3 = Interior Banding

Plus the alternative Banding Characters.

BandingCode

 

 

 

Banding code format.

Edge types: (L)ength, (W)idth, (R)adius and (A)ngle

Banding types: e(X)terior and (I)nterior

Example:1L15I1R6X1A12X - 1 Length 15" long with Interior, 1 6" Radius with eXterior, 1 Angled edge 12" long with eXterior

BandingFlags

 

 

 

 

 

 

Solid edge banding flags from a parts origin related to part edge as follows:

(edges code_1....code_n)

Example: 6NNENEN 6 sided part from face clockwise stroke:

N = None

I = Interior

E = Exterior

D = Door

FinishFace

The Face finish ID form material database.

FinishBack

The Back finish ID form material database.

FinishEdge

The Edge finish ID form material database.

Parameters

A list of System parameters associated with this part.
