Tool Catalog

Each Tool, when selected, will display a window to the left with the properties of that particular Tool listed.  Each of these properties are referred to as a "Field".

The following covers the Tool Properties Tab for Standard Tools. See the Standard Tool Shape Topic for information on Shaping Standard Tools.

The following is a description of each field including a reference to the importance of that field keyed by the number of stars preceding the item. Three stars () indicate the field is of critical importance, while one star, () indicates the fields of lesser importance.

General Tool Properties

  • Machine - All Tools

Select the Machine to assign this Tool to. This list also includes three standard choices plus a list of each individual Machine you have setup in your Machine Catalog.

The three standard choices are:

Any - Indicates that the active Tool is available to any Machine in your Machine Catalog.

Unavailable - Indicates that the Tool is unavailable (Not in Use).

Multiple - Indicates that the Tool is used on Multiple specific Machines in your Machine Catalog. After selecting this option you will  be presented with a list of all of your Machines and you can simply select which Machines this Tool is available for.

  • Name - All Tools

Enter a Name to identify the Tool. The name you give is for your reference only. In some posts the tool name will appear in a comment in the NC code next to the tool change.

  • Tool Number - All Tools

Critical for all tools except drills in a gang drill head. This is the ID that Intelli-CAM will use to match the Tool to the Machine in the Machine Catalog to the actual Tool on the Machine. For drills used in a gang drill head this value is ignored.

Some Machines (Post) do not use the Tool Number but instead use the Tool Name to uniquely identify the Tool.

  • Height Offset ID - All Tools

This is typically the same value as the tool number or the tool number + 100. This number refers to a preset value on the controller that defines the distance from the tip of the tool to the top of the spoil board or top of the Material.  This is a value set by the CNC operator .  It references the different heights of the tools above the work surface (In some posts this value is ignored).

  • Tool ID - All Tools

This is a unique ID Number assigned by the S2M Center. This value can be called from within your design package to assign a specific Tool to a specific Operation. For Example, within Cabinet Vision, in a User Created Standard you can use the Parameter ToolID equal to the Tools Tool ID number to make an assignment.

Tool Size Properties

  • Diameter - All Tools

The S2M Center will use this size to match tools to operations.  In addition it will determine for dado's if they are a single pass, rectangle, or pocket.  This diameter should match the diameter of the tool on the machine - however there are some instances where you may not want them to represent the actual size. One example; if you always wanted your dadoes to create a rectangle or single pass regardless of the actual size of the tool on the Machine. In this case you would enter a Tool Diameter that matched the dado width for a single pass or was smaller than the dado width for a rectangle. See the document on Tool Selection Logic for more information. With a Saw Tool the Diameter field defines the actual Diameter of the Saw.

  • Work Depth/Length - All Tools

Defines the height of the tool. Typically, this is for a visual in the Tool Shape Tab. This information is not normally used by the Post Processor as Tool Heights are typically set up at the Machine.

  • Drill to Shoulder - Vertical Drills

This will determine the final depth of any drill hole.  Be cautious setting this value to true since the drill tip has the potential to drill deeper than the shoulder.  This is a good option to use on a drill bit with a sharp drill cone angle and it is imperative the hole diameter at depth specified is the full diameter of the bit.  This is related to DRILL CONE ANGLE below.

  • Drill Cone Angle - Vertical Drills

This number is only important if you are drilling to the shoulder.  You have the option in Intelli-CAM when drilling, to reference the drill bit by the tip or by the shoulder.  A value of 0° or 180° here sets the drill tip and drill shoulder at the same value.  The smaller the drill cone angles the greater the gap between the drills tip and shoulder.  On most standard drill bits the drill cone angle is 118°.

Tool Selection Properties

  • Allow Auto Select

This option will keep the S2M Center from evaluating and assigning this Tool automatically using Automatic Tool Selection logic. This Tool will still be used if it is specifically assigned to an Operation via your CAD application.

  • Outline Tool - Flat Routers

The Outline tool is the tool used to cut all perimeter cuts in a nest.  It is necessary for one and ONLY ONE tool to be set as the Outline tool.  See the document on Tool Selection Logic for more information.

  • Down Shear/Up Shear - Flat Routers

This field assists the S2M Center in selecting the correct Tool for an operation depending on whether the Material properties requires the minimization of Face Chipping, Back Chipping, or both.

  • Through - Vertical Drills

This value will assist the S2M Center in selecting the right Tool for the Job.  If you have two identical Tools except for this value it will decide which Tool to use based on if the operation goes all the way through the Material or not.

  • Diameter Tolerance - Vertical and Horizontal Drills

This value allows a Diameter mismatch tolerance in case your Tool Diameter differs from the Geometry you would like to drill with the Tool. The most common use for this would be if you define your Drill Diameter in Metric but your geometry may be defined in Imperial.

  • Direction - Horizontal Drills

This option restricts a Horizontal Drill to moving in a certain direction to match the actual tool/machine restrictions.  This will assist the S2M Center in selecting the correct tool for an operation.  Some machines are orientated with the width along the long axis, while others are orientated with the width along the short axis. This allows Intelli-CAM to meet each requirement.  Allows saws, for instance, to operate only on the "X" axis. If no tool can be found to create the operation it will produce an error message during output.

Tool Movement Properties

  • Output Path - Flat Routers

This will determine if the path of the tool is described from the center or the edge of the tool. This, typically, should be set to "Offset" for Intelli-CAM output, Intelli-CAM favors the "Offset" path. "Offset" is the preferred output for the greatest level of accuracy after tools have been sharpened.  However, only some output methods support offset paths.  For these output methods the tool path should be set to "Center Line".  Check for your Machine's capabilities.

  • Rotate Clockwise - All Tools

This field in addition to the Material field "Climb Cut" will determine the Tool path direction.  Think of the router bit as a wheel.  If the path the wheel is following is the same direction, as a wheel would follow if rotated that direction, it is a "Climb Cut" otherwise, it is a "Conventional Cut".  An inside cut will always be in the opposite direction of an outside cut.  A "Conventional Cut" will always be in the opposite direction of a "Climb Cut".  A simple rule is: If the Material is set for a "Climb Cut" an outside pass would be in the same direction of Tool rotation.  Here is a chart to keep straight all the options: Remember the analogy to the wheel that will help keep the matter, including inside, outside, etc... straight.

  • Max Depth Per Pass - Vertical Drills & Flat Routers

For some CNC output, this will split an operation into multiple instructions so that a tool will make multiple passes to achieve the final depth.

This field should be set to 0 for a drill being used in a gang drill head unless your Post has been modified to support it.   Setting this value to 0 will ensure the correct routine is called in the post for the gang drill head.  This is a good option to use with large drill bits and narrow router bits where the depth of cut may require excess strain on the tool. For Drill tools any value other than 0 will call a Pecking rather than a Bore Cycle in your Post.

Drill operations terminology:

        1. Bore: The tool makes the complete operation in one cycle.

        2. Pecking: The tool creates the bore by making a series of plunges, each one deeper than the previous one, until the desired depth is reached.

      If you receiving a "Pecking Not Allowed" or "No Pecking Allowed" message during output then that means your Post does not support Pecking. You can contact Cabinet Vision to see if your Post can be modified to support Pecking.

  • Add to Through Bore/Cut - Flat Routers and Vertical and Horizontal Drills

This value is very important; the machine operator can be very particular about this option.  If the value is kept at the default (0) it may be a little difficult to separate the parts on your 1st nested sheet.  Adding a small value allows for a cut slightly deeper than the thickness of the sheet thusly allowing cleaner part separation from the original sheet, this will leave a trace of the path in the spoil board.

Tool Nesting Properties

Tool Feed Rate Properties

  • Feed Rate - Flat Routers and Vertical Drills

This determines how fast the Machine will try to move the Tool through the Material.  This is often a difficult field to find values for since we lack a standard for feed rate.  The value represents Inches per minute, or MM per Minute. This can be even further confusing since some posts expect a unit but then convert the unit in the post.  For example the MultiCAM post expects Inches Per Minute but converts the final output to Inches per Second.  As this process progresses expect to see a standardized unit for feed rate.  Until we get there you may have to experiment with this output. In addition the material will have an effect on the final feed rate.  When testing take into account the % feed rate property in the Material Catalog.

  • 1/4 Feed Rate - Flat Routers

This determines how fast the Machine will try to move the Tool through the Material at ¼” Deep.

  • 3/4 Feed Rate - Flat Routers

This determines how fast the Machine will try to move the Tool through the Material at ¾” Deep.

  • Descent Rate - All Tools

This will have the same unit of measurement as feed rate.

  • Spindle Speed - All Tools

This is the speed at which the tool will spin.  The unit is almost always RPM - Rotations per minute.  Setting a value of 0 here will usually cause an error in the NC Code.  A valid range would be about 4000 to 24000.  It would depend on the Machine and Tool's recommended specifications.  Some Machines will ignore this number and use a preset number for a Tool.

Tool CAD Properties

  • Pen - All Tools

Defines a pen color when viewing DXF output in AutoCAD® or other 3rd Party CAD/CAM applications.

  • Layer - All Tools

This is the Layer Name Operations utilizing this Tool will pass to your CAD/CAM package. You can use variables to define a layer name.  

Valid variables include:

  • {TS} = Tool Size
  • {DZ} = Tool Depth
  • {ZP} = Tool Z Position (Measured from work piece top/front face.)
  • {ZB} = Tool Z Position (Measured from work piece bottom/back face.)
  • {TC} = Tool Comp
  • {TN} = Tool Number
  • {PN} = Tool Pen Number
  • {FD} = Tool Feed Rate
  • {DR} = Tool Descent Rate.
