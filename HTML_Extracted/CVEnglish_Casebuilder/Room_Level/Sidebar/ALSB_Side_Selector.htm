Side Selector Section View - Section Face

When you first enter the Section Editor with a Standard Assembly you will be presented with the following sidebar view.

You can select any of the four Sides presented in the Side Selector. By selecting either of the Ends represented by the Side selector you will receive the following options:

  • End Type - You can define the Ends Type. There are eleven options here:

    1. Automatic - When Automatic is selected as the End Type you give Casebuilder control to determine what the End type is based on it’s condition. Basically this means it follows the logic of Unfinished End when against a Wall or other Assembly or Finished end when the End is exposed.

    2. Unfinished - Changes the End type to an Unfinished End following its Properties as defined in whichever Construction Method that is active. This is an Override to Casebuilder’s "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    3. Finished - Changes the End type to a Finished End following its Properties as defined in whichever Construction Method that is active. This is an Override to Casebuilder's "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    4. Applied Finished End - Changes the End type to an Applied Finished End following its Properties as defined in whichever Construction Method that is active. This is an Override to Casebuilder's "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    5. Applied Door - Changes the End type to an Applied Door Finished End following its Properties as defined in whichever Construction Method that is active, utilizing the selected Door Style for the Classes assigned End Panel. This is an Override to Casebuilder's "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    6. Panelized - Changes the End type to a Panelized Finished End following its Properties as defined in whichever Construction Method that is active, utilizing the selected Door Style for the Classes assigned End Panel. This is an Override to Casebuilder's "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    7. Extended – Changes the End type to an Extended Unfinished End following its Properties as defined in whichever Construction Method that is active. This is an Override to Casebuilder's "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    8. Return - Changes the End type to a Return Unfinished End following its Properties as defined in whichever Construction Method that is active. This is an Override to Casebuilder's "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    9. Filler - Changes the End Type to Filler. This is an Override to Casebuilder's "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    10. Skin - Changes the End Type to Skin. This is an Override to Casebuilder's "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    11. No End - Changes the End Type to No End. This is an Override to Casebuilder's "Automatic" End Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

By selecting the Back represented by the Side selector you will receive the following options:

  • Back Type - You can define the Backs Type. There are five options here:

    1. Automatic - When Automatic is selected as the Back Type you give Casebuilder control to determine what the Back type is based on it’s condition.

    2. Unfinished - Changes the Back type to an Unfinished Back following its Properties as defined in whichever Construction Method that is active. This is an Override to Casebuilder's "Automatic" Back Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    3. Finished - Changes the Back type to a Finished Back following its Properties as defined in whichever Construction Method that is active. This is an Override to Casebuilder's "Automatic" Back Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    4. Clipped Back Support - Changes the Back Type to a Clipped Back Support following its Properties as defined in whichever Construction Method that is active. This is an Override to Casebuilder's "Automatic" Back Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.

    5. No Back - Changes the Back Type to No Back. This is an Override to Casebuilder's "Automatic" Back Type and will stick with the Assembly under all conditions, even when Saved into your Assembly Catalog.
